# ⚾ 야구 주심 판정 게임 기획서

> 버전: v0.5
> 작성일: 2026-04-15
> 상태: 싱글플레이 완성 / 멀티플레이 완성

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 게임명 | 야구 주심 판정 게임 (가칭: **Strike Zone**) |
| 장르 | 스포츠 / 판단력 / 시뮬레이션 |
| 한 줄 설명 | 투수가 던진 공을 3D 시점으로 바라보며 스트라이크/볼을 판정하는 주심 체험 게임 |
| 플레이 인원 | 싱글플레이 + 멀티플레이 (동일 투구 판정 후 정답률 비교, 최대 10인) |
| 렌더링 방식 | Three.js (3D) |
| 서버 방식 | Firebase Realtime Database (멀티플레이 동기화) |
| 기록 저장 | 있음 (Firestore 연동) |

## 2. 게임 컨셉

플레이어는 **야구 주심**이 되어 포수 뒤에서 투구를 바라본다.
투수가 공을 던지면 3D 공의 궤적을 보고 스트라이크존 통과 여부를 판단하여
키보드/마우스로 판정 입력 후 정답과 비교해 점수를 얻는다.

### 핵심 재미 요소
- 공의 구종(직구, 커브, 슬라이더 등)에 따른 다양한 궤적
- 스트라이크존 경계를 아슬아슬하게 통과하는 공 → 판단의 긴장감
- 이닝/카운트 상황에 따른 압박감
- 판정 정확도에 따른 등급 및 랭킹 시스템
- **멀티플레이**: 같은 투구를 보고 각자 판정 → 정답률로 친구와 경쟁

## 3. 게임 규칙

### 3.1 기본 진행 구조
- 한 게임은 **9타자** 상대로 진행
- 타자마다 **랜덤 체형** 부여 → 체형에 따라 스트라이크존 높이/너비 자동 결정
- 삼진(스트라이크 3) 또는 볼넷(볼 4)이 되면 타자 교체 + **타자 카운트 +1**
- 9타자를 모두 상대하면 게임 종료

### 3.2 타자 체형 & 스트라이크존
- 타자 체형은 게임 시작 시 9명 전원 랜덤 생성
- 체형 파라미터: 키(소/중/대), 체형(마른/보통/다부짐), 타격 자세(스탠스 높이)
- 스트라이크존 = **홈플레이트 위 × 무릎~허리(유니폼 상단) 높이** → 체형에 따라 높이 범위 상이
- 타자 등장 시 체형 정보와 함께 스트라이크존 박스가 잠깐 표시(난이도에 따라 표시 시간 다름)

### 3.3 판정 방식
- 공이 **포수 미트에 들어간 후 3초 이내**에 판정 입력
- 판정: **S키 또는 우클릭 → 스트라이크** / **B키 또는 좌클릭 → 볼**
- 3초 내 미입력 시 자동 오심 처리 (가장 가까운 경계 기준 감점)

### 3.4 점수 시스템

| 상황 | 점수 |
|------|------|
| 정확한 판정 (스트라이크/볼 모두) | +100점 |
| 오판정 | −(보더라인까지의 거리 비례) 점 |
| 경계 안쪽 5% 이내 정확 판정 (보더라인) | +150점 (보너스) |
| 연속 정답 콤보 | ×1.1 ~ ×2.0 배율 적용 |

**오판정 감점 계산식**
- 공의 실제 위치와 스트라이크존 경계 사이 거리를 0~1로 정규화
- 거리가 멀수록 감점 증가: 최대 −100점, 경계선 직전 오판정은 최소 −5점

### 3.5 타자 교체 조건

| 조건 | 결과 |
|------|------|
| 스트라이크 3 (삼진) | 타자 아웃 → 다음 타자 |
| 볼 4 (볼넷) | 타자 출루 → 다음 타자 |
| 9타자 완료 | 게임 종료 |

## 5. 게임 모드

| 모드 | 인원 | 설명 |
|------|------|------|
| 연습 모드 | 1인 | 스트라이크존 항상 표시, 판정 직후 정답 공개, 기록 저장 안 함 |
| 일반 모드 | 1인 | 9타자 정규 진행, 기록 저장 |
| **멀티 배틀** | **2~10인** | **같은 투구 시퀀스를 동시에 판정 → 종료 후 정답률 비교** |

### 멀티 배틀 상세
- 방장이 방을 생성하면 초대 링크 또는 방 코드(6자리)로 참가, 최대 10인
- 모든 플레이어가 준비 완료 시 게임 시작
- 동일한 투구 시퀀스(같은 난수 시드)를 공유 → 모두 같은 공, 같은 타자 체형을 봄
- 각자 독립적으로 판정 입력 (서로의 판정은 게임 중 비공개)
- 모든 구수 종료 후 결과 화면에서 플레이어별 정확도/점수/등급 순위 비교

## 6.4 투구 리플레이 시스템

### 개요
- 모든 투구는 투구 파라미터(구종, 폼, 구속, 무브먼트, 홈플레이트 통과 좌표)를 저장
- 하단 리플레이 목록에서 투구 번호 클릭 → 해당 투구 재생

### 리플레이 화면 기능

| 기능 | 설명 |
|------|------|
| 공 궤적 재생 | 해당 투구의 3D 궤적을 처음부터 다시 재생 |
| 궤적 라인 표시 | 공이 지나간 경로를 선으로 시각화 |
| 스트라이크존 표시 | 해당 타자의 존 박스를 반투명으로 표시 |
| 통과 지점 마커 | 홈플레이트를 통과한 정확한 좌표에 점/원 표시 |
| 판정 결과 표시 | 실제 정답(S/B)과 플레이어 판정 비교 |
| 재생 속도 조절 | 0.25× / 0.5× / 1× 슬로우모션 선택 |
| 시점 전환 | 리플레이 중 포수 뒤 / 정면 / 측면 시점 전환 가능 |

### 데이터 구조 (클라이언트 메모리)
```json
{
  "pitchIndex": 3,
  "pitchType": "slider",
  "pitcherForm": "sidearm",
  "speed": 128,
  "movementX": -0.18,
  "movementY": -0.05,
  "plateX": 0.21,
  "plateZ": 0.65,
  "isStrike": true,
  "playerCall": "ball",
  "correct": false,
  "scoreChange": -72
}
```

## 7. 3D 씬 구성 (Three.js)

| 요소 | 내용 |
|------|------|
| 카메라 | 포수 뒤 주심 시점 (홈플레이트 기준 기본 위치) |
| 마운드 | 투수 마운드 3D 기하 |
| 홈플레이트 | 3D 오각형 플레이트 |
| 스트라이크존 | 반투명 박스 (타자 체형 기반 자동 크기 조정) |
| 타자 | 랜덤 체형 3D 캐릭터, 좌타/우타 배치 |
| 투수 | 랜덤 투구폼 4종 애니메이션 |
| 공 | Three.js Sphere, 베지어 곡선 궤적 |
| 조명 | 낮 경기 기준 Directional Light + Ambient |
| 배경 | 야구장 외야 스탠드 |

### 투수 투구폼 (랜덤 4종)

| 폼 | 특징 | 공 릴리즈 포인트 |
|----|------|----------------|
| 오버핸드 (Overhand) | 정통파, 수직 무브먼트 강조 | 머리 위 |
| 쓰리쿼터 (3/4) | 가장 일반적인 폼 | 귀 옆 대각선 |
| 사이드암 (Sidearm) | 수평 무브먼트 강조, 좌우 횡변화 | 어깨 높이 옆 |
| 언더핸드 (Underhand) | 하체에서 올라오는 궤적, 상승 무브먼트 | 허리 아래 |

- 타자 등장마다 투구폼 랜덤 결정 (같은 타자에 대해서는 고정)
- 폼에 따라 공의 초기 방향벡터와 무브먼트 특성이 달라짐

### 카메라 조작

| 키 | 동작 |
|----|------|
| 기본 위치 | 포수 뒤 주심 시점 (고정) |
| `C` 키 | 카메라 조작 모드 ON/OFF 토글 |
| `↑ ↓` 또는 `W S` | 카메라 상하 이동 (소폭) |
| `← →` 또는 `A D` | 카메라 좌우 이동 (소폭) |
| `Q / E` | 카메라 각도 회전 (좌/우 틸트) |
| `R` | 카메라 기본 위치로 초기화 |

- 조작 범위는 주심 위치 기준 제한된 범위 내로 클램핑
- 카메라 조작 중에는 판정 입력 비활성화 (투구 중에는 자동 비활성)
- 투구 시작 시 카메라 조작 모드 자동 해제

### 공 궤적 구현 방식
- 투구 시작점(마운드)에서 끝점(홈플레이트)까지 **베지어 곡선** 으로 이동
- 구종 + 투구폼 조합으로 고유한 무브먼트 파라미터 적용
- 공 속도: 난이도에 따라 80~150km/h 범위

## 9. 게임 흐름

### 싱글플레이
```
게임 시작
  │
  ▼
난이도 선택 (루키 / 아마추어 / 프로 / 메이저)
  │
  ▼
9명 타자 체형 랜덤 생성 (키/체형/스탠스)
  │
  ▼
타자 등장 → 스트라이크존 잠깐 표시 → 숨김
투수 투구폼 랜덤 결정 (오버핸드/쓰리쿼터/사이드암/언더핸드)
  │
  ▼
투구 애니메이션 시작 (3D 공 날아옴)
  │
  ▼
공 포수 미트 도착 → 3초 카운트다운 시작
  │
  ▼
플레이어 판정 입력 (S키 / B키) 또는 타임아웃
  │
  ├── 정답 → +100점 (보더라인 +150) + 콤보 증가 + 피드백
  └── 오답/타임아웃 → 거리 비례 감점 + 피드백
  │
  ▼
투구 데이터 리플레이 목록에 저장
  │
  ▼
카운트 업데이트 (B/S) → 삼진/볼넷 시 타자 교체
  │
  ▼
[9타자 완료?]
  ├── 아니오 → 다음 투구
  └── 예 → 결과 화면 → Firestore 저장 → 랭킹 표시
```

### 멀티 배틀
```
방 생성 (방장, 난이도 설정) / 방 참가 (6자리 코드)
  │
  ▼
대기실 (참가자 목록 실시간 표시, 최대 10인, 모두 준비 시 시작)
  │
  ▼
공통 난수 시드 공유 → 동일한 타자 9명 + 투구 시퀀스 생성
  │
  ▼
투구 진행 (모든 플레이어 동시, 각자 독립 판정)
  → Firebase Realtime DB에 판정 저장
  │
  ▼
[9타자 완료?]
  ├── 아니오 → 다음 투구
  └── 예 → 결과 비교 화면 (플레이어별 정확도/점수/등급 순위)
             → Firestore에 멀티 기록 저장
```

## 10. 난이도 시스템

| 레벨 | 투구 속도 | 사용 구종 | 보더라인 비율 | 존 표시 시간 |
|------|-----------|-----------|--------------|-------------|
| 루키 | 80~100km/h | 직구, 체인지업 | 10% | 3초 |
| 아마추어 | 100~120km/h | + 투심, 슬라이더 | 20% | 2초 |
| 프로 | 120~140km/h | + 커브 | 35% | 1초 |
| 메이저 | 140~150km/h | 전 구종 (스플리터 포함) | 50% | 0.5초 |

- **보더라인 비율**: 스트라이크존 경계 ±5% 이내에 의도적으로 걸치는 투구 비율
- **존 표시 시간**: 타자 등장 시 스트라이크존 박스를 보여주는 시간 (이후 숨김)
