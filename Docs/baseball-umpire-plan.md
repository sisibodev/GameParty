## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 게임명 | 야구 주심 판정 게임 (가칭: **Strike Zone**) |
| 장르 | 스포츠 / 판단력 / 시뮬레이션 |
| 한 줄 설명 | 투수가 던진 공을 3D 시점으로 바라보며 스트라이크/볼을 판정하는 주심 체험 게임 |
| 플레이 인원 | 싱글플레이 + 멀티플레이 (동일 투구 판정 후 정답률 비교, 최대 10인) |

| 기록 저장 | 있음 (Firestore 연동) |

## 2. 게임 컨셉

플레이어는 **야구 주심**이 되어 포수 뒤에서 투구를 바라본다.
투수가 공을 던지면 3D 공의 궤적을 보고 스트라이크존 통과 여부를 판단하여
키보드/마우스로 판정 입력 후 정답과 비교해 점수를 얻는다.

### 핵심 재미 요소
- 공의 구종(직구, 커브, 슬라이더 등)에 따른 다양한 궤적
- 스트라이크존 경계를 아슬아슬하게 통과하는 공 → 판단의 긴장감
- 이닝/카운트 상황에 따른 압박감
- 판정 정확도에 따른 등급 및 랭킹 시스템
- **멀티플레이**: 같은 투구를 보고 각자 판정 → 정답률로 친구와 경쟁

## 3. 게임 규칙

### 3.1 기본 진행 구조
- 한 게임은 **총 30구** 진행
- **5구마다 타자 교체** → 총 6명의 타자 순서로 진행
- **10구마다 투수 투구폼 변경** → 총 3번 폼 전환 (중복 가능)
- 30구를 모두 소화하면 게임 종료

### 3.2 타자 체형 & 스트라이크존
- 타자 체형은 게임 시작 시 6명 전원 랜덤 생성
- 체형 파라미터: 키(소/중/대), 체형(마른/보통/다부짐), 타격 자세(스탠스 높이)
- 스트라이크존 = **홈플레이트 위 × 무릎~허리(유니폼 상단) 높이** → 체형에 따라 높이 범위 상이
- 타자 등장 시 체형 정보와 함께 스트라이크존 박스가 **왼쪽 고정 패널에 항상 표시** (투구 화면 가리지 않음)
- KBO ABS 기준 적용: 신장의 **56.35%를 상단**, **27.64%를 하단**으로 스트라이크존 높이 계산

### 3.3 판정 방식
- 공이 **포수 미트에 들어간 후 3초 이내**에 판정 입력
- 판정: **S키 또는 우클릭 → 스트라이크** / **B키 또는 좌클릭 → 볼**
- 공은 미트에 도착하는 순간 **즉시 사라짐** → 잔상 없음, 판정 난이도 유지
- 3초 내 미입력 시 자동 오심 처리 (가장 가까운 경계 기준 감점)

### 3.4 점수 시스템

| 상황 | 점수 |
|------|------|
| 정확한 판정 (스트라이크/볼 모두) | +100점 |
| 오판정 | −(보더라인까지의 거리 비례) 점 |
| 경계 안쪽 5% 이내 정확 판정 (보더라인) | +150점 (보너스) |
| 연속 정답 콤보 | ×1.1 ~ ×2.0 배율 적용 |

**오판정 감점 계산식**
- 공의 실제 위치와 스트라이크존 경계 사이 거리를 0~1로 정규화
- 거리가 멀수록 감점 증가: 최대 −100점, 경계선 직전 오판정은 최소 −5점

### 3.6 스트라이크존 판정 로직 (KBO ABS 기준)

공의 **가장자리(반지름 포함)** 기준으로, **3개 면 중 2개 이상 통과 시 스트라이크** 판정.

| 체크포인트 | 위치 | 좌우 기준 |
|-----------|------|----------|
| 앞면 (Front) | 홈플레이트 앞 Z축 | 홈플레이트 너비 + 양끝 2cm 확대 적용 |
| 중간면 (Middle) | 홈플레이트 중간 Z축 | 홈플레이트 너비 + 양끝 2cm 확대 적용 |
| 끝면 (End) | 홈플레이트 끝 Z축 | 홈플레이트 너비 그대로 (확대 없음) |

**판정 규칙**
- 판정 기준: 공의 **가장자리(중심 ± 반지름)** 가 존 범위에 조금이라도 걸치면 해당 면 통과
- **3개 면 중 2개 이상 통과 → 스트라이크**
- **1개 이하 통과 → 볼**
- 끝면 상하 기준: 앞면/중간면보다 1.5cm 낮게 설정 (공의 낙하 보정)

**스트라이크존 상하 기준 (타자 신장 기반)**
- 상단 = 신장 × 56.35%
- 하단 = 신장 × 27.64%

**리플레이 시각화**
- 각 면마다 공이 **실제로 걸친(충돌한) 영역만** 빨간색으로 표시
- 통과하지 않은 면은 반투명 박스 그대로 유지 (색상 없음)
- 통과한 면 수를 카운트하여 판정 결과 함께 표시

### 3.5 타자 교체 조건

| 조건 | 결과 |
|------|------|
| 5구 소화 | 다음 타자로 교체 |
| 30구 완료 | 게임 종료 |

## 5. 게임 모드

| 모드 | 인원 | 설명 |
|------|------|------|
| 연습 모드 | 1인 | 스트라이크존 항상 표시, 판정 직후 정답 공개, 기록 저장 안 함 |
| 일반 모드 | 1인 | 30구 / 6타자 정규 진행, 기록 저장 |
| **멀티 배틀** | **2~10인** | **같은 30구 시퀀스를 동시에 판정 → 종료 후 정답률 비교** |

### 멀티 배틀 상세
- 방장이 방을 생성하면 초대 링크 또는 방 코드(6자리)로 참가, 최대 10인
- 모든 플레이어가 준비 완료 시 게임 시작
- 동일한 투구 시퀀스(같은 난수 시드)를 공유 → 모두 같은 공, 같은 타자 체형을 봄
- 각자 독립적으로 판정 입력 (서로의 판정은 게임 중 비공개)
- **실시간 점수 순위** 좌측 상단 패널에 표시, 투구마다 순위 변동 업데이트
- 모든 구수 종료 후 결과 화면에서 플레이어별 정확도/점수/등급 순위 비교

### 6.2 판정 피드백 UI
- 판정 직후 **화면 중앙**에 크게 표시: **"STRIKE!"** / **"BALL!"**
- 정답 여부: ✅ 정확 / ❌ 오심 + 감점 표시
- 보더라인 정확 판정 시: 🔥 **"BORDERLINE!"** 특수 연출

### 6.3 판정 결과 존 표시 (중앙 오버레이)
- 판정 후 **화면 중앙에** 2D 존 결과 뷰 표시
- 존 박스 비율은 **타자 체형에 맞는 실제 종횡비** 적용 (정사각형 고정 X)
- 공의 중심점 위치를 존 위에 마커로 표시
- 중간면/끝면 체크포인트 통과 여부 색상으로 구분

### 6.4 게임 종료 화면
- 총 판정 수, 정답 수, 정확도(%)
- 구종별 / 타자별 정확도 통계
- 최고 콤보, 오심 횟수
- 등급 (S/A/B/C/D)
- Firestore에 기록 저장 후 랭킹 표시

## 6.5 투구 리플레이 시스템

### 개요
- 모든 투구는 투구 파라미터(구종, 폼, 구속, 무브먼트, 홈플레이트 통과 좌표)를 저장
- 하단 히스토리에서 투구 번호 클릭 → **게임 일시 중단** 후 리플레이 재생
- 리플레이 종료 시 게임 재개

### 리플레이 카메라 연출 (3단계 시퀀스)

영상 참고 기반으로 다음 순서로 카메라가 자동 연출됨:

**1단계 — 정면 추적 (주심 시점)**
- 포수 뒤 주심 시점에서 공의 궤적을 따라 슬로우모션 재생
- 공이 위에서 아래로 떨어지는 궤적 라인(흰색) 표시
- 스트라이크존 반투명 박스 표시

**2단계 — 사선 시점 전환 (공 도착 직후)**
- 카메라가 홈플레이트 측면 비스듬한 각도로 자동 이동
- **중간면(앞면)** 체크포인트: 통과 시 초록, 미통과 시 빨간 반투명 면으로 표시
- **끝면(뒷면)** 체크포인트: 동일 방식으로 표시
- 공의 중심점이 각 면을 통과했는지 마커(노란 점)로 시각화

**3단계 — 완전 측면 (90도 회전)**
- 카메라가 홈플레이트 정측면으로 최종 이동
- 공의 궤적 라인을 측면에서 확인 (빨간 수직 드롭 라인)
- 스트라이크존 앞면/뒷면 경계를 측면에서 확인
- 판정 결과 텍스트 표시: **"중심점 기준 [스트라이크/볼] 판정"**

### 리플레이 조작

| 기능 | 설명 |
|------|------|
| 재생 속도 | 0.25× / 0.5× / 1× 선택 |
| 단계 수동 전환 | ← → 키로 1~3단계 직접 이동 |
| 존 표시 토글 | Z키로 스트라이크존 박스 표시/숨김 |
| 리플레이 종료 | ESC 또는 닫기 버튼 → 게임 재개 |

### 데이터 구조 (클라이언트 메모리)
```json
{
  "pitchIndex": 3,
  "pitchType": "slider",
  "pitcherForm": "sidearm",
  "speed": 128,
  "movementX": -0.18,
  "movementY": -0.05,
  "frontPlane":  { "passX": 0.23, "passZ": 0.68, "hit": true  },
  "midPlane":    { "passX": 0.21, "passZ": 0.65, "hit": true  },
  "endPlane":    { "passX": 0.19, "passZ": 0.58, "hit": false },
  "planeHitCount": 2,
  "isStrike": true,
  "playerCall": "ball",
  "correct": false,
  "scoreChange": -72
}
```
- `hit`: 해당 면에서 공 가장자리가 존 범위에 걸쳤는지 여부
- `planeHitCount`: 2 이상이면 스트라이크
- 리플레이에서 `hit: true`인 면만 **충돌 영역을 빨간색**으로 렌더링

## 7. 3D 씬 구성 (Three.js)

| 요소 | 내용 |
|------|------|
| 카메라 | 포수 뒤 주심 시점 (홈플레이트 기준 기본 위치) |
| 마운드 | 투수 마운드 3D 기하 |
| 홈플레이트 | 3D 오각형 플레이트 |
| 스트라이크존 | 반투명 박스 (타자 체형 기반 자동 크기 조정) |
| 타자 | 랜덤 체형 3D 캐릭터, 좌타/우타 배치 |
| 투수 | 랜덤 투구폼 4종 애니메이션 |
| 공 | Three.js Sphere, 베지어 곡선 궤적 |
| 조명 | 낮 경기 기준 Directional Light + Ambient |
| 배경 | 야구장 외야 스탠드 |

### 투수 투구폼 (랜덤 4종)

| 폼 | 특징 | 공 릴리즈 포인트 |
|----|------|----------------|
| 오버핸드 (Overhand) | 정통파, 수직 무브먼트 강조 | 머리 위 |
| 쓰리쿼터 (3/4) | 가장 일반적인 폼 | 귀 옆 대각선 |
| 사이드암 (Sidearm) | 수평 무브먼트 강조, 좌우 횡변화 | 어깨 높이 옆 |
| 언더핸드 (Underhand) | 하체에서 올라오는 궤적, 상승 무브먼트 | 허리 아래 |

- **10구마다 투구폼 변경** (1~10구, 11~20구, 21~30구 각 구간마다 랜덤 폼 결정, 중복 허용)
- 폼에 따라 공의 초기 방향벡터와 무브먼트 특성이 달라짐

### 카메라 조작

| 키 | 동작 |
|----|------|
| 기본 위치 | 포수 뒤 주심 시점 (고정) |
| `C` 키 | 카메라 조작 모드 ON/OFF 토글 |
| `↑ ↓` 또는 `W S` | 카메라 상하 이동 (소폭) |
| `← →` 또는 `A D` | 카메라 좌우 이동 (소폭) |
| `Q / E` | 카메라 각도 회전 (좌/우 틸트) |
| `R` | 카메라 기본 위치로 초기화 |

- 조작 범위는 주심 위치 기준 제한된 범위 내로 클램핑
- 카메라 조작 중에는 판정 입력 비활성화 (투구 중에는 자동 비활성)
- 투구 시작 시 카메라 조작 모드 자동 해제

### 공 궤적 구현 방식
- 투구 시작점(마운드)에서 끝점(홈플레이트)까지 **베지어 곡선** 으로 이동
- 구종 + 투구폼 조합으로 고유한 무브먼트 파라미터 적용
- 공 속도: 난이도에 따라 80~150km/h 범위

## 9. 게임 흐름

### 싱글플레이
```
게임 시작
  │
  ▼
난이도 선택 (루키 / 아마추어 / 프로 / 메이저)
  │
  ▼
6명 타자 체형 랜덤 생성 (키/체형/스탠스)
투구폼 구간별 결정: 1~10구 폼A / 11~20구 폼B / 21~30구 폼C (랜덤, 중복 가능)
  │
  ▼
타자 등장 → 왼쪽 패널에 체형/존 정보 표시 (고정)
  │
  ▼
투구 애니메이션 시작 (3D 공 날아옴)
  │
  ▼
공 포수 미트 도착 → 공 즉시 사라짐 → 3초 카운트다운 시작
  │
  ▼
플레이어 판정 입력 (S키 / B키) 또는 타임아웃
  │
  ├── 정답 → +100점 (보더라인 +150) + 콤보 증가 + 중앙 피드백
  └── 오답/타임아웃 → 거리 비례 감점 + 중앙 피드백
  │
  ▼
중앙에 존 결과 뷰 표시 (타자 체형 비율 적용)
투구 데이터 하단 히스토리에 저장
  │
  ▼
[5구 단위 타자 교체? / 10구 단위 투구폼 변경?]
  │
  ▼
[30구 완료?]
  ├── 아니오 → 다음 투구
  └── 예 → 결과 화면 → Firestore 저장 → 랭킹 표시
```

### 멀티 배틀
```
방 생성 (방장, 난이도 설정) / 방 참가 (6자리 코드)
  │
  ▼
대기실 (참가자 목록 실시간 표시, 최대 10인, 모두 준비 시 시작)
  │
  ▼
공통 난수 시드 공유 → 동일한 타자 6명 + 30구 시퀀스 생성
  │
  ▼
투구 진행 (모든 플레이어 동시, 각자 독립 판정)
  → 판정마다 Firebase Realtime DB 업데이트
  → 좌측 상단 실시간 순위 패널 갱신
  │
  ▼
[30구 완료?]
  ├── 아니오 → 다음 투구
  └── 예 → 결과 비교 화면 (플레이어별 정확도/점수/등급 순위)
             → Firestore에 멀티 기록 저장
```

## 10. 난이도 시스템

| 레벨 | 투구 속도 | 사용 구종 | 보더라인 비율 |
|------|-----------|-----------|--------------|
| 루키 | 80~100km/h | 직구, 체인지업 | 10% |
| 아마추어 | 100~120km/h | + 투심, 슬라이더 | 20% |
| 프로 | 120~140km/h | + 커브 | 35% |
| 메이저 | 140~150km/h | 전 구종 (스플리터 포함) | 50% |

- **보더라인 비율**: 스트라이크존 경계 ±5% 이내에 의도적으로 걸치는 투구 비율
- 타자 체형 정보 및 존 박스는 **왼쪽 패널에 항상 고정 표시** (난이도 무관)
