## 1. 게임 개요

| 항목 | 내용 |

| 게임명 | 야구 주심 판정 게임 (가칭: **Strike Zone**) |

| 한 줄 설명 | 투수가 던진 공을 3D 시점으로 바라보며 스트라이크/볼을 판정하는 주심 체험 게임 |

## 2. 게임 컨셉

플레이어는 **야구 주심**이 되어 포수 뒤에서 투구를 바라본다.
투수가 공을 던지면 3D 공의 궤적을 보고 스트라이크존 통과 여부를 판단하여
키보드/마우스로 판정 입력 후 정답과 비교해 점수를 얻는다.

### 핵심 재미 요소
- 공의 구종(직구, 커브, 슬라이더 등)에 따른 다양한 궤적
- 스트라이크존 경계를 아슬아슬하게 통과하는 공 → 판단의 긴장감
- 이닝/카운트 상황에 따른 압박감
- 판정 정확도에 따른 등급 및 랭킹 시스템
- **멀티플레이**: 같은 투구를 보고 각자 판정 → 정답률로 친구와 경쟁

## 3. 게임 규칙

### 3.1 기본 진행 구조
- 한 게임은 **총 30구** 진행
- **5구마다 타자 교체** → 총 6명의 타자 순서로 진행
- **10구마다 투수 투구폼 변경** → 총 3번 폼 전환 (중복 가능)
- 30구를 모두 소화하면 게임 종료

### 3.2 타자 체형 & 스트라이크존
- 타자 체형은 게임 시작 시 6명 전원 랜덤 생성
- 체형 파라미터: 키(소/중/대), 체형(마른/보통/다부짐), 타격 자세(스탠스 높이)
- 스트라이크존 = **홈플레이트 위 × 무릎~허리(유니폼 상단) 높이** → 체형에 따라 높이 범위 상이
- 타자 등장 시 체형 정보와 함께 스트라이크존 박스가 **왼쪽 고정 패널에 항상 표시** (투구 화면 가리지 않음)
- KBO ABS 기준 적용: 신장의 **56.35%를 상단**, **27.64%를 하단**으로 스트라이크존 높이 계산

### 3.3 판정 방식
- 공이 **포수 미트에 들어간 후 3초 이내**에 판정 입력
- 판정: **S키 또는 → 키 → 스트라이크** / **B키 또는 ← 키 → 볼**
- 공은 미트에 도착하는 순간 **즉시 사라짐** → 잔상 없음, 판정 난이도 유지
- 3초 내 미입력 시 자동 오심 처리 (가장 가까운 경계 기준 감점)

### 3.4 점수 시스템

| 상황 | 점수 |
|------|------|
| 정확한 판정 (스트라이크/볼 모두) | +100점 |
| 오판정 | −(보더라인까지의 거리 비례) 점 |
| 경계 안쪽 5% 이내 정확 판정 (보더라인) | +150점 (보너스) |
| 연속 정답 콤보 | ×1.1 ~ ×2.0 배율 적용 |

**오판정 감점 계산식**
- 공의 실제 위치와 스트라이크존 경계 사이 거리를 0~1로 정규화
- 거리가 멀수록 감점 증가: 최대 −100점, 경계선 직전 오판정은 최소 −5점

### 3.6 스트라이크존 판정 로직 (KBO ABS 기준)

공의 **가장자리(반지름 포함)** 기준으로, **3개 면 중 2개 이상 통과 시 스트라이크** 판정.

| 체크포인트 | 위치 | 좌우 기준 |
|-----------|------|----------|
| 앞면 (Front) | 홈플레이트 앞 Z축 (z=+0.13) | 홈플레이트 너비 + 양끝 2cm 확대 적용 |
| 중간면 (Middle) | 홈플레이트 중간 Z축 (z=0) | 홈플레이트 너비 + 양끝 2cm 확대 적용 |
| 끝면 (End) | 홈플레이트 끝 Z축 (z=−0.13) | 홈플레이트 너비 그대로 (확대 없음) |

**판정 규칙**
- 판정 기준: 공의 **가장자리(중심 ± 반지름)** 가 존 범위에 조금이라도 걸치면 해당 면 통과
- **3개 면 중 2개 이상 통과 → 스트라이크**
- **1개 이하 통과 → 볼**
- 끝면 상하 기준: 앞면/중간면보다 1.5cm 낮게 설정 (공의 낙하 보정)

**스트라이크존 상하 기준 (타자 신장 기반)**
- 상단 = 신장 × 56.35%
- 하단 = 신장 × 27.64%

### 3.5 타자 교체 조건

| 조건 | 결과 |
|------|------|
| 5구 소화 | 다음 타자로 교체 |
| 30구 완료 | 게임 종료 |

## 4. 구종 목록

총 **10종**. 난이도에 따라 직구 + 변화구 N종이 매 게임 랜덤으로 선택됨.

| 구종 | 코드 | 구속 비율 | 특징 |
|------|------|-----------|------|
| 직구 (4-Seam Fastball) | `fastball` | 100% (기준) | 직선 궤적, 기본 |
| 투심 패스트볼 | `two_seam` | 93~97% | 약간의 수평 무브먼트 |
| 싱커 | `sinker` | 92~97% | 하강 무브먼트, 땅볼 유도 |
| 커터 | `cutter` | 88~93% | 짧은 수평 꺾임 |
| 체인지업 | `changeup` | 80~87% | 속도 변화로 눈속임, 하강 |
| 슬라이더 | `slider` | 85~91% | 좌우로 휘는 궤적 |
| 스위퍼 | `sweeper` | 80~86% | 슬라이더보다 큰 수평 휩쓸기 |
| 커브 | `curve` | 78~85% | 위에서 아래로 크게 꺾임 |
| 스플리터 | `splitter` | 87~93% | 급격히 아래로 떨어짐 |
| 포크볼 | `forkball` | 82~88% | 후반 급강하, 스플리터보다 늦게 떨어짐 |

**구속 비율**: 직구 구속 × 비율 = 해당 구종 구속. 체인지업/커브가 항상 직구보다 느리도록 보장됨.

## 5. 게임 모드

| 모드 | 인원 | 설명 |
|------|------|------|
| 연습 모드 | 1인 | 스트라이크존 항상 표시, 판정 직후 정답 공개, 기록 저장 안 함 |
| 일반 모드 | 1인 | 30구 / 6타자 정규 진행, 기록 저장 |
| **멀티 배틀** | **2~10인** | **같은 30구 시퀀스를 동시에 판정 → 종료 후 정답률 비교** |

### 멀티 배틀 상세
- 방장이 방을 생성하면 초대 링크 또는 방 코드(6자리)로 참가, 최대 10인
- 모든 플레이어가 준비 완료 시 게임 시작
- 동일한 투구 시퀀스(같은 난수 시드)를 공유 → 모두 같은 공, 같은 타자 체형을 봄
- 각자 독립적으로 판정 입력 (서로의 판정은 게임 중 비공개)
- **투구마다 실시간 점수 동기화** — Firebase RTDB `updateLiveScore` 로 매 판정 직후 갱신
- **실시간 점수 순위** 좌측 상단 패널에 표시, 투구마다 순위 변동 업데이트
- 모든 구수 종료 후 결과 화면에서 플레이어별 정확도/점수/등급 순위 비교

### 6.2 판정 피드백 UI
- 판정 직후 **화면 중앙**에 크게 표시: **"STRIKE!"** / **"BALL!"**
- 정답 여부: ✅ 정확 / ❌ 오심 + 감점 표시
- 보더라인 정확 판정 시: 🔥 **"BORDERLINE!"** 특수 연출
- **리플레이 열람 중에는 피드백 UI 자동 숨김**

### 6.3 판정 결과 존 표시 (중앙 오버레이)
- 판정 후 **화면 중앙에** 2D 존 결과 뷰 표시
- 존 박스 비율은 **타자 체형에 맞는 실제 종횡비** 적용 (정사각형 고정 X)
- 존 박스 · 공 위치 모두 동일 좌표계(`toSvgX/Y`)로 매핑 — 실제 판정 존과 시각이 정확히 일치
- 공의 중심점 위치를 존 위에 마커로 표시
- 보더라인 투구 시 존 테두리가 노란 점선으로 전환
- 중간면/끝면 체크포인트 통과 여부 색상으로 구분

### 6.4 게임 종료 화면
- 총 판정 수, 정답 수, 정확도(%)
- 구종별 / 타자별 정확도 통계 + SVG 차트
- 최고 콤보, 오심 횟수
- 등급 (S/A/B/C/D)
- **투구 히스토리 테이블**: 구수별 구종/구속/판정/결과 + ▶ 리플레이 버튼

- **멀티 배틀 결과화면**도 싱글과 동일 UI: 플레이어 순위표 + 내 등급/점수 + 구종별 정확도 + 투구 기록 테이블 + ▶ 리플레이

## 6.6 투구 리플레이 시스템

### 개요
- 모든 투구는 투구 파라미터(구종, 폼, 구속, 무브먼트, 홈플레이트 통과 좌표, 3면 통과 여부)를 저장
- **게임 중**: `↑` 키로 마지막 투구 즉시 리플레이. 리플레이 열람 중 피드백 UI 숨김
- **결과 화면**: 투구 히스토리 테이블의 ▶ 버튼으로 원하는 투구 리플레이 가능

### 리플레이 카메라 연출 (3단계 자동 시퀀스)

**1단계 — 정면 추적 (주심 시점)**
- 포수 뒤 주심 시점에서 공의 궤적을 따라 슬로우모션 재생
- 공이 **홈플레이트 앞 13cm → 중앙 → 뒤 13cm(z=−0.30)까지 3면 전부 통과**
- 공 궤적 라인(노란색) 표시
- 스트라이크존 반투명 박스 표시

**2단계 — 사선 시점 전환 (공 도착 직후)**
- 카메라가 홈플레이트 측면 비스듬한 각도로 자동 이동
- **공(구체) 숨김** → 3개 ABS 면 각각의 충돌 위치에 마커로 대체 표시
  - **충돌한 면(hit)**: 주황색 링 + 점 (불투명)
  - **미통과 면(miss)**: 회색 링 + 점 (반투명)
- KBO ABS 3면 반투명 박스(앞/중/끝) 동시 표시
- 수동 이동(`← →`)으로 단계 전환 시 2단계 이후엔 공이 보이지 않음

**3단계 — 완전 측면 (90도 회전)**
- 카메라가 홈플레이트 정측면으로 최종 이동
- 빨간 수직 드롭 라인으로 공의 낙하 위치 표시
- 스트라이크존 앞면/뒷면 경계를 측면에서 확인

### 리플레이 컨트롤 패널 (우측 오버레이)

| 영역 | 내용 |
|------|------|
| 투구 정보 | 구수 (#N), 구종명(한글), 구속(km/h), BORDERLINE 뱃지 |
| 판정 결과 | 실제 판정 / 내 판정 / 정오답 / ABS 통과면 수 |
| 단계 인디케이터 | 1~3단계 도트 + 이전/다음 버튼 |
| 재생 속도 | 0.25× / 0.5× / 1× 토글 |
| 다시 재생 | 1단계부터 처음부터 재생 |
| 단축키 힌트 | `← →` 단계 이동 · `↓` / `ESC` 닫기 |

### 3면 마커 데이터 구조 (PitchParams)
```typescript
frontPlaneHit?: boolean   // z=+0.13 면 통과 여부
midPlaneHit?:   boolean   // z=0    면 통과 여부
endPlaneHit?:   boolean   // z=−0.13 면 통과 여부
planeHitCount?: number    // 0~3, 2 이상 = 스트라이크
```

## 7. 3D 씬 구성 (Three.js)

| 요소 | 내용 |
|------|------|
| 카메라 | 포수 뒤 주심 시점 (홈플레이트 기준 기본 위치) |
| 마운드 | 투수 마운드 3D 기하 |
| 홈플레이트 | 3D 오각형 플레이트 |
| 스트라이크존 | 반투명 박스 (타자 체형 기반 자동 크기 조정) |
| 타자 | GLB 모델 (`baseball-idle.glb`), BatterProfile 체형(키/체형)에 따른 scale 자동 조정, 좌타/우타 배치 |
| 투수 | GLB 모델 (`baseball-pitching.glb`), 오버핸드 기반 4종 투구폼 (오버핸드 완성 / 3쿼터·사이드·언더 제작 예정) |
| 공 | Three.js Sphere, CubicBezierCurve3 궤적 (구종별 고유 무브먼트), 블롭 그림자(CircleGeometry) — 높이에 따라 크기·불투명도 자동 조절 |
| 조명 | 낮 경기 기준 Directional Light + Ambient |
| 배경 | 야구장 외야 스탠드 |

### 투수 투구폼 (랜덤 4종)

| 폼 | 특징 | 공 릴리즈 포인트 |
|----|------|----------------|
| 오버핸드 (Overhand) | 정통파, 수직 무브먼트 강조 | 머리 위 |
| 쓰리쿼터 (3/4) | 가장 일반적인 폼 | 귀 옆 대각선 |
| 사이드암 (Sidearm) | 수평 무브먼트 강조, 좌우 횡변화 | 어깨 높이 옆 |
| 언더핸드 (Underhand) | 하체에서 올라오는 궤적, 상승 무브먼트 | 허리 아래 |

- **10구마다 투구폼 변경** (1~10구, 11~20구, 21~30구 각 구간마다 랜덤 폼 결정, 중복 허용)

### 투구폼 구현 방식

- 애니메이션 파일 위치: `public/games/baseball-umpire/animations/`
- **오버핸드**: `baseball-pitching.glb` — 기본 GLB 애니메이션 그대로 사용
- **쓰리쿼터 / 사이드암 / 언더핸드**: 오버핸드 GLB를 기반으로 별도 GLB 제작 예정
  - 각 폼별로 릴리즈 포인트(position) 및 팔 각도(bone rotation)를 조정하여 변형

### 타자 애니메이션 & Scale 조정

- 애니메이션 파일: `public/games/baseball-umpire/animations/baseball-idle.glb`
- BatterProfile 체형 파라미터에 따른 Three.js scale 자동 적용

| BatterHeight | scale.y 배율 |
|---|---|
| short (165cm) | 0.92 |
| medium (178cm) | 1.00 (기준) |
| tall (190cm) | 1.07 |

| BatterBuild | scale.x / scale.z 배율 |
|---|---|
| slim | 0.92 |
| normal | 1.00 (기준) |
| stocky | 1.08 |

- 좌타(`isLefty=true`)는 X축 반전(`scale.x *= -1`) 적용

### 공 궤적 구현 방식
- **CubicBezierCurve3** (3차 베지어): 제어점 2개로 구종별 고유 무브먼트 구현 (기본 모드, 현재 활성화)
- **물리 시뮬레이션** (`buildPhysicsCurve`): RK4 수치적분, 중력 · 공기저항 · 마그누스 효과 → 구현 완료, 추후 UI 노출 예정
- 끝점 z=−0.30 → 3개 ABS 판정 면(z=+0.13, 0, −0.13)을 모두 통과
- `PITCH_SPEED_RATIO`: 구종별 직구 대비 구속 배율 → 체인지업이 직구보다 빨라지는 버그 원천 차단
- `PITCH_BREAK`: `{t1, y1, t2, y2}` 형태로 구종별 곡선 형태 정의

## 10. 난이도 시스템

| 레벨 | 투구 속도 | 변화구 종수 | 보더라인 비율 | 존 미리보기 |
|------|-----------|-------------|--------------|-------------|
| 루키 | 90~110km/h | 직구 + 2종 | 10% | 3,000ms |
| 아마추어 | 110~130km/h | 직구 + 3종 | 20% | 2,000ms |
| 프로 | 130~150km/h | 직구 + 4종 | 35% | 1,000ms |
| 메이저 | 150~160km/h | 직구 + 5종 | 50% | 500ms |

**변화구 선택 방식**
- 게임 시작 시 `BREAKING_BALL_POOL` (9종) 에서 **Fisher-Yates partial shuffle** 로 N종 랜덤 선택
- 멀티플레이에서는 공유 시드(`initialSeed`) 기반 Seeded RNG 사용 → 모든 플레이어 동일한 구종 세트
- 선택된 구종은 게임 내내 고정 (투구마다 재선택 없음)
- 모드 선택 화면에 `직구 + ? × N 매 게임 랜덤 선택` 형태로 표시

**구속 계산**
- 기본 구속: 난이도 범위 내 랜덤 (직구 기준)
- 각 구종 구속 = 직구 구속 × `PITCH_SPEED_RATIO[pitchType]`
- 체인지업은 직구보다 항상 13~20% 느림 (구속 역전 불가)
