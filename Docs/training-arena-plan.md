### 상세 스펙

#### 1. 초기 스탯
| 대상 | hp | str | agi | int | luk |
|------|----|-----|-----|-----|-----|
| 플레이어    | 1  | 1   | 1   | 1   | 1   |
| NPC (모두) | 1  | 1   | 1   | 1   | 1   |

제거 상수: `INITIAL_PLAYER_STAT_POINTS`, `PLAYER_EXTRA_STAT_POINTS`.
변경 상수: `INITIAL_NPC_STAT` = 1 (v0.5.0: 10→1), `NPC_BASE_GROWTH` = 1.
제거 페이지: `StatAllocPage`.

#### 2. 라운드 시작 스탯 가챠 ✓
라운드마다 전원 1장씩 — 랜덤 스탯 1종 + 등급별 랜덤 수치.

| 등급 | 확률 | 수치 범위 | 기대값 (100연 시) |
|------|------|-----------|------------------|
| C    | 40%  | 1–2  | 40회 |
| B    | 24%  | 2–4  | 24회 |
| A    | 14%  | 4–6  | 14회 |
| S    |  9%  | 6–9  | 9회 |
| SS   |  7%  | 9–13 | 7회 |
| SSS  |  6%  | 13–18 | 6회 |

코드: `GACHA_PROBABILITIES`, `GACHA_STAT_RANGES` (constants.ts)
플레이어는 연출 확인, NPC 결과는 MatchPreviewPage에서 공개.

#### 3. 전술 카드 — 고정 8종 상시 선택지 ✓
플레이어는 매 전투마다 8종 중 1장을 고르거나 "사용 안 함" 선택. 쿨다운/소모 없음.

| # | ID | 카드명 | 효과 | 발동 시점 | 변수 포인트 |
|---|----|--------|------|-----------|-------------|
| 1 | `first_strike` | 선제 공격 | ATB +60으로 시작 | 전투 시작 | 스피드 열세 극복 |
| 2 | `barrier` | 방벽 전개 | 첫 피격 데미지 -70% | 첫 피격 | 버스트 딜러 카운터 |
| 3 | `ambush` | 기습 | 첫 공격 크리 확정 + ×1.3 | 내 첫 공격 | 한 방 역전 |
| 4 | `mana_burst` | 마나 폭주 | 마나 +100%, 리젠 +50% (5턴) | 전투 시작 | 스킬 난사형 |
| 5 | `curse` | 저주 | 상대 ATK -30%, DEF -20% (4턴) | 전투 시작 | 강캐 약화 |
| 6 | `potion` | 회복 물약 | HP 30% 이하 시 50% 회복 (1회) | 자동 트리거 | 장기전 생존 |
| 7 | `insight` | 간파 | 상대 회피율 0, 내 크리 +30% (5턴) | 전투 시작 | 회피 캐릭 카운터 |
| 8 | `last_stand` | 최후의 일격 | HP 20% 이하: ATK +100%, SPD +30% | 자동 활성 | 빈사 역전 |

코드: `tacticCards.ts`, `BattleCharState.tactic`, `applyTactic()` (battleEngine.ts)
사용 규칙: MatchPreviewPage에서 선택, 상대 스탯·스킬·아이템을 보고 카운터 조합 고르기.

#### 4. 골드·상점·아이템 ⚠
**골드 획득 시스템 ✓**

라운드 종료 시 플레이어 성적에 따른 일괄 지급 (MVP/순회 단위 아님).

| 최종 결과 | 골드 |
|----------|------|
| 우승 (1라운드 통합 최강) | 400 |
| 준우승 (결승 탈락) | 200 |
| 토너먼트 아웃 | 100 |
| 본선 아웃 (조별 리그) | 60 |
| 예선 아웃 | 30 |
| 라이벌전 승리 시 | ×2 배율 |

코드: `GOLD_BY_RESULT` (constants.ts), `rewardEngine.ts`

**상점 & 아이템** ⚠
- ✓ 구현됨: 스탯 아이템 (common/rare/hero/legend, 영구 효과 합산)
- ✓ 구현됨: 일부 전투 아이템 (독 단검, 마력 봉인, 흡혈 반지, 불굴의 투지)
- ✓ 구현됨: 황금 장갑 (골드 ×1.3 배율)
- ✓ 인벤토리 최대 100칸 (누적 보관)
- ✓ 상점 리롤 로직 구현 (비용 30 골드) — ⏳ UI 버튼 숨김 (보류)

**아이템 목록** (items.ts)
- A. 스탯 아이템(20개): 5 등급 × 5 스탯 (common +3 ~ legend +30)
- B. 전투 아이템(4개): 독 단검(rare), 마력 봉인(hero), 흡혈 반지(hero), 불굴의 투지(legend)
- C. 유틸 아이템(1개): 황금 장갑(rare, 골드 배율)

#### 5. 라이벌 시스템 ✓
- 캐릭터 선택 직후 랜덤 NPC 3명 지정 (RIVAL_COUNT = 3)
- **+5 보너스 스탯을 매 라운드 누적 분배** (RIVAL_STAT_PER_ROUND = 5)
  - 예: 1라운드 +5, 2라운드 누적 +10, 3라운드 누적 +15 … (라이벌은 후반으로 갈수록 강해짐)
- 라이벌전 승리 시 골드 ×2 배율 (RIVAL_GOLD_MULTIPLIER = 2)
- 라이벌 탈락해도 회차 내 유지, 다음 게임에서 재지정
- ⏳ 라이벌전 UI 표식(🔥) — 구현 상태 확인 필요

#### 6. 대전 프리뷰 공개 정보
`MatchPreviewPage`에서 상대의 스탯 전체, 스킬 목록(쿨다운·우선순위 포함), 아이템 인벤토리를 공개. 이 정보로 전술 카드 선택.

#### 7. 스킬 학습 시스템 (수련 방식) ✓
"강탈"이 아닌 "학습/수련" — 상대 스킬을 복제하며, 상대는 그 스킬을 그대로 보유.

- 라운드 종료 시 랜덤 스킬 보상 제거
- 각 전투 승리 시 상대 스킬 중 **1개 선택해 학습 시작 또는 건너뛰기**
- 상대는 스킬을 잃지 않음 (복제 방식)

**학습 대기 기간 (전투 회수 기반 차감)** ✓

| 스킬 등급 | 학습 소요 전투 수 |
|-----------|-------------------|
| common | 1 |
| rare   | 3 |
| hero   | 5 |
| legend | 10 |

코드: `SKILL_LEARN_TURNS` (constants.ts), `PendingSkill` (types.ts)

규칙:
- ✓ 학습 중 스킬은 슬롯 점유·전투 사용 불가 (pending 상태)
- ✓ 본인 참여 전투 종료마다 -1 차감 (관전 경기는 차감 안 함)
- ✓ 0 도달 시 사용 가능 상태로 전환
- ✓ 여러 스킬 동시 학습 가능 (슬롯 허략 시)
- ✓ 최대 슬롯 5개 (MAX_SKILL_SLOTS = 5)
- ✓ 초과 시 교체 UI 제공 (학습 중 스킬도 교체 대상)

**타입 정의** (types.ts)
```ts
export interface PendingSkill {
  skillId:        string
  turnsRemaining: number  // 0 = 학습 완료 (본인 참여 전투 종료마다 -1)
}
```

### 게임 루프 (v0.4.1+) ✓
```
[슬롯 선택 또는 신규 생성]
  ↓
[캐릭터 선택 + 라이벌 3명 자동 지정]
  ↓
반복 (라운드 N):
  [상점] (2라운드부터) → [스탯 가챠] (전원 1장 C~SSS 범위)
  ↓
  [예선 → 본선(조별 리그 더블 엘리미) → 토너먼트 (16강→결승)]
    각 매치:
      1. MatchPreviewPage: 상대 스탯/스킬/아이템 공개 + 전술 카드 선택
      2. BattlePage: 자동 전투 시뮬레이션 + 로그 표시
      3. MatchResultPage: 결과 표시, 승리 시 스킬 학습 선택
  ↓
[라운드 우승 또는 탈락]
  → 골드 지급 (성적별)
  → 스탯 자동 분배 (미구현)
  → 기록 저장 (RunRecord)
  ↓
다음 라운드로 (무한 반복)
또는 슬롯 복귀
```

### v0.4.1+ 추가 시스템

#### ✓ 캐릭터 해금
- 시작 시 해금된 캐릭터 5명: id `[1, 2, 3, 4, 5]` (INITIAL_UNLOCKED_CHAR_IDS)

#### ✓ 본선 더블 엘리미네이션 (조별 리그)
4인 1조 × 8개 조 (GROUP_COUNT=8, GROUP_SIZE=4)

| 경기 | 타입 | 설명 |
|------|------|------|
| m1, m2 | `initial` | A vs B, C vs D |
| m3 | `winners` | 1경기 승자끼리 → 승자 본선 확정 |
| m4 | `losers` | 1경기 패자끼리 → 패자 탈락 확정 |
| m5 | `decider` | m3 패자 vs m4 승자 → 승자 본선 진출 |

각 조에서 2명씩 16강 진출 (16×8 = 총 32명).
코드: `runGroup()`, `GroupResult` (tournamentEngine.ts)

#### ✓ 나가기 버튼 (게임플레이 화면 고정)

### v0.4.2 전투 시스템 디테일 (코드 검증)
#### ATB 시스템 ✓
- 최대 게이지: 100 (MAX_ATB_GAUGE = 100)
- 행동 순서: 속도(SPD) 기반, 매 턴 SPD × 배율만큼 게이지 증가
- 초기 마나: 최대 마나의 50% (INITIAL_MANA_RATIO = 0.5)
- 마나 리젠: maxMana × (maxHp + atk) × 0.001 (MANA_REGEN_COEFF)

#### INT 시스템 ✓
- 스킬 쿨다운 감소: INT 성장치 40당 -1턴, 최대 -3턴 (INT_CD_REDUCTION_PER = 40)
- 속도 보너스: 일부 직업에서 INT → SPD 계수 적용 (int_to_spd, mage: 1.2)

#### AGI 시스템 ✓
- 회피 상쇄: 공격자 SPD × 0.3만큼 수비자 EVA 차감 (AGI_EVA_COUNTER_COEFF = 0.3)
- 빠른 공격자가 회피율을 상쇄하는 메커니즘

#### HP 회복 ✓
- 매 턴 자연 회복: maxHp × 0.8% (HP_REGEN_PER_TURN_RATIO = 0.008)
- 경기 간 HP 회복: maxHp × 30% (INTER_MATCH_HP_REGEN_RATIO = 0.30)

#### 전술 카드 효과 (battleEngine.ts 구현) ✓
| 카드ID | 코드 구간 |
|--------|----------|
| `first_strike` | gauge 초기값 60 설정 |
| `barrier` | 첫 피격 데미지 -70% (×0.3 유지) |
| `ambush` | 첫 공격 크리 확정 + ×1.3 데미지 |
| `mana_burst` | 마나 +100% 및 리젠 +50% (5턴) |
| `curse` | 상대 ATK -30%, DEF -20% (4턴) |
| `potion` | HP ≤30% 시 자동 50% 회복 (1회) |
| `insight` | 상대 회피율 0, 내 크리 +30% (5턴) |
| `last_stand` | HP ≤20% 시 ATK +100%, SPD +30% |

## 1. 게임 개요

### 한 줄 설명
캐릭터 하나를 골라 스탯을 찍고, 가챠로 다른 경쟁자들을 강화시킨 뒤, 예선부터 토너먼트까지 올라가 우승을 노리는 **육성형 자동 전투 시뮬레이션**.

### 핵심 특징
- 100% 자동 전투 — 플레이어는 **육성 / 분배 / 선택**만 담당
- 가챠로 뽑은 카드가 **경쟁자들까지 강화**시키는 독특한 구조 (본인 캐릭터가 뽑히면 본인도 강해짐)
- 매 회차마다 우승한 캐릭터의 스킬을 흡수하며 **장기적 성장** 누적
- 플레이어 개입 없이 AI끼리 돌리는 **순수 시뮬레이션 모드** 별도 제공

### 장르
육성 / 자동 전투 / 가챠 / 토너먼트 시뮬레이션

### 플레이 인원
싱글 (1인)

### 렌더링 방식
PixiJS (2D) — 캐릭터 표시, 전투 연출, 가챠 연출

### 서버 방식
없음 (클라이언트 완결)

### 기록 저장
**Firestore** — 플레이어 프로필, 누적 스탯, 보유 스킬, 우승까지 걸린 회차, 대회 이력

## 3. 핵심 게임 루프

```
[플레이어 모드]
캐릭터 선택
  → 초기 스탯 분배 (플레이어 기본 스탯 포인트 **20점** 지급)
  → 가챠 100연 (경쟁자 + 본인 강화)
  → 예선 → 본선 → 토너먼트
  → 보상 수령:
      - 랜덤 스탯 자동 부여 (단계별, 어느 스탯인지 표시)
      - 플레이어 추가 스탯 +10 직접 분배
  → 스킬 습득 선택 (스킬 3개 후보 중 1개, 넘기기 가능)
      - 스킬 슬롯 8개 초과 시 기존 스킬 1개와 교체
  → 다음 회차 (누적 진행)
```

목표: **최소 회차로 우승하기** (회차 수를 DB에 기록)

## 4. 스탯 시스템

### 4.1 성장 스탯 (5종)

모든 캐릭터 공통, 초기값 **1로 통일**. 가챠 / 대회 보상으로 상승.

| 스탯 | 영향 |
|------|------|
| 체력 | HP, 방어력, 마나 리젠률 |
| 힘 | 공격력, 방어력 |
| 민첩 | 공격속도, 회피, 크리티컬 |
| 지력 | 마나, 스킬 쿨타임, 마나 리젠률 |
| 행운 | 회피, 크리티컬, 스킬 발동률, 적극성 |

- **최대치**: 각 스탯 **9999**
- 랜덤 부여 시 해당 스탯이 최대치를 초과하면 **다른 스탯으로 재분배**

### 4.2 전투 스탯 (성장 스탯으로부터 파생)

| 전투 스탯 | 계산 기반 |
|------|------|
| HP | 기본값 + 체력 (0이 되면 패배) |
| 공격력 | 기본값 + 힘 |
| 방어력 | 기본값 + 힘 + 체력 |
| 공격 속도 | 기본값 + 민첩 (행동 순서 결정) |
| 크리티컬 | 기본값 + 행운 + 민첩 |
| 회피 | 기본값 + 행운 + 민첩 |
| 마나 | 기본값 + 지력 (스킬 소모 자원, 리젠률은 체력+지력) |
| 적극성 | 기본값 + 행운 (내 턴에 스킬을 사용할지 결정) |

### 4.3 캐릭터 기본 전투 스탯 (100점 베이스)

모든 캐릭터는 아래 기본값에 **추가 100점을 컨셉에 맞게 배분**해서 시작.

| 스탯 | 기본값 |
|------|------|
| HP | 20 |
| 공격력 | 10 |
| 방어력 | 5 |
| 공격 속도 | 1 |
| 크리티컬 | 10 |
| 회피 | 0 |
| 마나 | 10 |
| 적극성 | 5 |

> 예: 중장갑 캐릭터라면 HP / 방어력 쪽에 많이 배분.
> 구체 배분표는 [캐릭터 기획서](./training-arena-character-plan.md) 참조.

## 5. 가챠 시스템

### 5.1 기본 규칙
- 대회 시작 **직전**에 가챠 진행
- **100연 가챠** (연차 수는 밸런스 단계에서 재검토)
- 카드 1장마다 **랜덤 등급** 결정 → 등급에 따라 해당 캐릭터에게 **랜덤 스탯 증가**

### 5.2 등급 테이블 (확정)

| 등급 | 증가 스탯 | 출현 확률 |
|------|----------|----------|
| C | +1 | **40%** |
| B | +2 | **24%** |
| A | +3 | **14%** |
| S | +4 | **9%** |
| SS | +5 | **7%** |
| SSS | +6 | **6%** |

100연 기대값: SSS 6회 / SS 7회 / A급 이상 약 36회.

### 5.3 본인 캐릭터 규칙
- 가챠에서 **본인 캐릭터가 뽑히면** → 본인 캐릭터도 해당 등급만큼 스탯 상승
- 즉, 가챠는 "경쟁자 강화 vs 본인 강화"의 운 요소

## 6. 대회 구조

### 6.1 진행 단계
```
예선 → 본선 → 토너먼트 → 우승
```

### 6.2 HP 회복 규칙
- 각 **단계 이행 시**: HP 최대치 회복
- 단계 내 **경기와 경기 사이**: `회복량 = 최대 HP × 30%` (최대 HP 상한)
  - 예) 최대 100, 현재 30 → 60 (30 회복)
  - 예) 최대 100, 현재 80 → 100 (20만 회복)
- 토너먼트 단계 내(16강 → 결승)도 동일 규칙 (4강 → 결승 특전 회복 없음)

### 6.3 단계별 보상

| 결과 | 랜덤 스탯 자동 부여 |
|------|------|
| 우승 | +10 |
| 준우승 (결승 탈락) | +7 |
| 토너먼트 탈락 (16강·8강·4강) | +3 |
| 본선 탈락 (조별 리그) | +2 |
| 예선 탈락 | +1 |
| 예선 탈락자 중 **30%** (다크호스 룰) | +5 추가 |

- 랜덤 스탯은 **자동으로 5종 중 하나에 배정** (플레이어가 선택 불가, 결과만 표시)
- 추가로 **플레이어 본인에게는 원하는 곳에 분배 가능한 스탯 +10** 별도 지급
- 토너먼트 결과에서 **16강·8강·4강·준우승·우승** 단계별로 명확히 표시

### 6.4 스킬 슬롯 규칙 (확정)
- 플레이어 스킬 슬롯 최대 **8개**
- 미달 시: 스킬 3개 후보 중 1개 선택하거나 **넘기기**
- 초과 시 (현재 8개): 기존 스킬 1개를 **교체** (삭제하고 새 스킬 획득)
- 넘기기를 선택하면 이번 회차 스킬 습득 없이 다음 단계로

## 11. 저장 데이터 전략 (확정)

### 11.1 저장 슬롯 시스템
- 플레이어당 **최대 3개 저장 슬롯**
- 각 슬롯은 **독립된 플레이 기록** (선택 캐릭터, 누적 스탯, 보유 스킬, 회차 번호)
- 예선·본선·토너먼트 어느 단계에서 탈락해도 **계속 성장 누적** (게임오버 없음)
- 슬롯이 모두 차 있고 새 게임을 시작하려면 → 덮어쓸 슬롯 1개를 선택하여 초기화

### 11.2 저장소 이원화

| 데이터 | 저장소 | 이유 |
|--------|--------|------|
| 로그인 인증 / 허용 이메일 확인 | Firebase Auth | 기존 플랫폼 공통 |
| 슬롯 프로필 (선택 캐릭터, 누적 스탯, 스킬, 회차) | **Firestore** | 기기 간 동기화 |
| 대회 **요약** (우승자, 회차 번호, 보상 내역) | **Firestore** | 장기 이력 |
| 전투 **상세 로그** (턴별 액션) | **IndexedDB** (브라우저 로컬) | 용량 대비 무료, 동기화 불필요 |
| 시뮬레이션 모드 대회 상세 | **IndexedDB** | 동일 |

### 11.3 Firestore 스키마 (요약)

```
/users/{userId}/slots/{slotId}  (slotId = 1 | 2 | 3)
  - characterId: string
  - cumulativeStats: { 체력, 힘, 민첩, 지력, 행운 }
  - skills: string[]           (보유 스킬 ID, 최대 8개)
  - currentRound: number
  - bestClearRound: number | null
  - createdAt, updatedAt: timestamp

/users/{userId}/slots/{slotId}/tournaments/{tournamentId}
  - round: number
  - winner: charId
  - stage: "완료"
  - playerResult: "우승" | "토너먼트" | "본선" | "예선" | "예선탈락"
  - rewards: { stat, amount, acquiredSkillId }
```

### 11.4 IndexedDB 스키마 (전투 로그)

```
DB: battleGrandPrix
  ObjectStore: matchLogs
    - matchId (key)
    - tournamentId, slotId, stage
    - log: [ { turn, actor, action, damage, ... } ]
    - createdAt
```

비용 관점: Firestore 무료 티어(1GB, 20K write/day) 범위 내. IndexedDB는 브라우저 로컬로 비용 0.
