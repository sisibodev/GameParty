### 핵심 변화
1. **초기 스탯 고정** — 플레이어 1/1/1/1/1, NPC 10/10/10/10/10. 초기 배분 UI(StatAllocPage) 제거.
2. **라운드 시작 가챠** — 모든 캐릭터가 매 라운드 1장씩 C~SSS 등급의 랜덤 스탯 카드를 획득. (기존 100연 가챠 대체)
3. **전술 카드 시스템(신규)** — 6~8장 풀에서 라운드당 3장 보유, 전투 전 상대를 보고 0~1장 사용해 전투에 개입.
4. **골드·상점·아이템(신규)** — 승리로 골드 누적, 라운드 사이 상점에서 등급별 스탯/전투 아이템 구매. NPC도 성적별 차등 확률로 아이템 획득.
5. **라이벌 시스템(신규)** — 3명 지정, 매 라운드 +5 보너스 스탯, 라이벌전 승리 시 골드 ×2.
6. **대전 전 상대 정보 완전 공개** — 스탯/스킬/아이템 모두 표시.
7. **스킬 획득 방식 변경** — 라운드 종료 시 랜덤 선택 제거, **승리 시 상대방 스킬 1개 강탈**.
8. **점수 & 기록 시스템(신규)** — 우승 시 라운드·전적·스탯·스킬로 점수 산출, 도감에 캐릭터별 최고 점수 등록, 슬롯별 플레이 기록을 점수 내림차순으로 저장하고 상세 열람.

#### 2. 라운드 시작 스탯 가챠
라운드마다 전원 1장씩 — 랜덤 스탯 1종 + 등급별 랜덤 수치.

| 등급 | 확률 | 수치 범위 |
|------|------|-----------|
| C    | 40%  | 1–2  |
| B    | 24%  | 2–4  |
| A    | 14%  | 4–6  |
| S    |  9%  | 6–9  |
| SS   |  7%  | 9–13 |
| SSS  |  6%  | 13–18 |

플레이어는 연출 확인, NPC 결과는 대전 프리뷰에서 공개.

<span style="color:#3b82f6">🔵 **CHANGED** — 전술 카드는 랜덤 지급이 아닌 **고정 8종 상시 선택지**로 변경. 플레이어는 매 전투마다 8종 중 1장을 고르거나 "사용 안 함" 선택. 소모 개념 없음.</span>

#### 3. 전술 카드 — 고정 8종 상시 선택지
| # | 카드 | 효과 | 발동 시점 | 변수 포인트 |
|---|------|------|-----------|-------------|
| 1 | 선제 공격 | 내 ATB +60으로 시작 | 전투 시작 | 스피드 열세 극복 |
| 2 | 방벽 전개 | 첫 피격 데미지 -70% | 첫 피격 | 버스트 딜러 카운터 |

| 4 | 마나 폭주 | 마나 초기치 +100%, 마나 리젠 +50% (5턴) | 전투 시작 | 스킬 난사형 |

| 7 | 간파 | 상대 회피율 0, 내 크리율 +30% (5턴) | 전투 시작 | 회피/민첩 캐릭 카운터 |
| 8 | 최후의 일격 | HP 20% 이하에서 ATK +100%, SPD +30% | 트리거 | 빈사 역전 |

### v0.4.1 추가 시스템

#### 🟢 NEW — 캐릭터 해금
- 시작 시 해금된 캐릭터 5명: id `[1, 2, 3, 4, 5]` (상수 `INITIAL_UNLOCKED_CHAR_IDS`).
- 게임 플레이 중 **대회에 등장한 NPC 캐릭터**(내가 직접 대전하거나 대진표에 올라온 캐릭터)를 해금.
- 해금 상태는 `localStorage`로 플레이어 전역 관리 (슬롯 간 공유). 신규 해금 배지(`newChars`) 표시.
- 캐릭터 선택 화면에서 해금된 캐릭터만 선택 가능, 미해금은 실루엣 처리.
- 시뮬레이션 모드는 해금 영향을 받지 않음(전체 풀에서 관전).

#### 🟢 NEW — 본선 더블 엘리미네이션 (조별 리그)
본선은 4인 1조 × 8개 조 구성. 각 조 내부는 다음 4경기로 진행:

| 경기 | `GroupMatchType` | 설명 |
|------|-----------------|------|
| 1경기 | `initial` | A vs B, C vs D (초기 대진) |
| 2경기 | `winners` | 1경기 승자끼리 → 승자는 **본선 통과 확정** |
| 3경기 | `losers`  | 1경기 패자끼리 → 패자는 **탈락 확정** |
| 4경기 | `decider` | winners 패자 vs losers 승자 → 승자 본선 통과 |

- 각 조에서 **2명씩 16강 진출**.
- 패자부활을 통해 1패 후에도 반등할 기회 제공. 운빨 1라운드 극복 가능.

## 1. 게임 개요

### 한 줄 설명
캐릭터 하나를 골라 스탯을 찍고, 가챠로 다른 경쟁자들을 강화시킨 뒤, 예선부터 토너먼트까지 올라가 우승을 노리는 **육성형 자동 전투 시뮬레이션**.

### 핵심 특징
- 100% 자동 전투 — 플레이어는 **육성 / 분배 / 선택**만 담당
- 가챠로 뽑은 카드가 **경쟁자들까지 강화**시키는 독특한 구조 (본인 캐릭터가 뽑히면 본인도 강해짐)
- 매 회차마다 우승한 캐릭터의 스킬을 흡수하며 **장기적 성장** 누적
- 플레이어 개입 없이 AI끼리 돌리는 **순수 시뮬레이션 모드** 별도 제공

### 장르
육성 / 자동 전투 / 가챠 / 토너먼트 시뮬레이션

### 플레이 인원
싱글 (1인)

### 렌더링 방식
PixiJS (2D) — 캐릭터 표시, 전투 연출, 가챠 연출

### 서버 방식
없음 (클라이언트 완결)

### 기록 저장
**Firestore** — 플레이어 프로필, 누적 스탯, 보유 스킬, 우승까지 걸린 회차, 대회 이력

## 3. 핵심 게임 루프

```
[플레이어 모드]
캐릭터 선택
  → 초기 스탯 분배 (플레이어 기본 스탯 포인트 **20점** 지급)
  → 가챠 100연 (경쟁자 + 본인 강화)
  → 예선 → 본선 → 토너먼트
  → 보상 수령:
      - 랜덤 스탯 자동 부여 (단계별, 어느 스탯인지 표시)
      - 플레이어 추가 스탯 +10 직접 분배
  → 스킬 습득 선택 (스킬 3개 후보 중 1개, 넘기기 가능)
      - 스킬 슬롯 8개 초과 시 기존 스킬 1개와 교체
  → 다음 회차 (누적 진행)
```

목표: **최소 회차로 우승하기** (회차 수를 DB에 기록)

## 4. 스탯 시스템

### 4.1 성장 스탯 (5종)

모든 캐릭터 공통, 초기값 **1로 통일**. 가챠 / 대회 보상으로 상승.

| 스탯 | 영향 |
|------|------|
| 체력 | HP, 방어력, 마나 리젠률 |
| 힘 | 공격력, 방어력 |
| 민첩 | 공격속도, 회피, 크리티컬 |
| 지력 | 마나, 스킬 쿨타임, 마나 리젠률 |
| 행운 | 회피, 크리티컬, 스킬 발동률, 적극성 |

- **최대치**: 각 스탯 **9999**
- 랜덤 부여 시 해당 스탯이 최대치를 초과하면 **다른 스탯으로 재분배**

### 4.2 전투 스탯 (성장 스탯으로부터 파생)

| 전투 스탯 | 계산 기반 |
|------|------|
| HP | 기본값 + 체력 (0이 되면 패배) |
| 공격력 | 기본값 + 힘 |
| 방어력 | 기본값 + 힘 + 체력 |
| 공격 속도 | 기본값 + 민첩 (행동 순서 결정) |
| 크리티컬 | 기본값 + 행운 + 민첩 |
| 회피 | 기본값 + 행운 + 민첩 |
| 마나 | 기본값 + 지력 (스킬 소모 자원, 리젠률은 체력+지력) |
| 적극성 | 기본값 + 행운 (내 턴에 스킬을 사용할지 결정) |

### 4.3 캐릭터 기본 전투 스탯 (100점 베이스)

모든 캐릭터는 아래 기본값에 **추가 100점을 컨셉에 맞게 배분**해서 시작.

| 스탯 | 기본값 |
|------|------|
| HP | 20 |
| 공격력 | 10 |
| 방어력 | 5 |
| 공격 속도 | 1 |
| 크리티컬 | 10 |
| 회피 | 0 |
| 마나 | 10 |
| 적극성 | 5 |

> 예: 중장갑 캐릭터라면 HP / 방어력 쪽에 많이 배분.
> 구체 배분표는 [캐릭터 기획서](./training-arena-character-plan.md) 참조.

## 5. 가챠 시스템

### 5.1 기본 규칙
- 대회 시작 **직전**에 가챠 진행
- **100연 가챠** (연차 수는 밸런스 단계에서 재검토)
- 카드 1장마다 **랜덤 등급** 결정 → 등급에 따라 해당 캐릭터에게 **랜덤 스탯 증가**

### 5.2 등급 테이블 (확정)

| 등급 | 증가 스탯 | 출현 확률 |
|------|----------|----------|
| C | +1 | **40%** |
| B | +2 | **24%** |
| A | +3 | **14%** |
| S | +4 | **9%** |
| SS | +5 | **7%** |
| SSS | +6 | **6%** |

100연 기대값: SSS 6회 / SS 7회 / A급 이상 약 36회.

### 5.3 본인 캐릭터 규칙
- 가챠에서 **본인 캐릭터가 뽑히면** → 본인 캐릭터도 해당 등급만큼 스탯 상승
- 즉, 가챠는 "경쟁자 강화 vs 본인 강화"의 운 요소

## 6. 대회 구조

### 6.1 진행 단계
```
예선 → 본선 → 토너먼트 → 우승
```

### 6.2 HP 회복 규칙
- 각 **단계 이행 시**: HP 최대치 회복
- 단계 내 **경기와 경기 사이**: `회복량 = 최대 HP × 30%` (최대 HP 상한)
  - 예) 최대 100, 현재 30 → 60 (30 회복)
  - 예) 최대 100, 현재 80 → 100 (20만 회복)
- 토너먼트 단계 내(16강 → 결승)도 동일 규칙 (4강 → 결승 특전 회복 없음)

### 6.3 단계별 보상

| 결과 | 랜덤 스탯 자동 부여 |
|------|------|
| 우승 | +10 |
| 준우승 (결승 탈락) | +7 |
| 토너먼트 탈락 (16강·8강·4강) | +3 |
| 본선 탈락 (조별 리그) | +2 |
| 예선 탈락 | +1 |
| 예선 탈락자 중 **30%** (다크호스 룰) | +5 추가 |

- 랜덤 스탯은 **자동으로 5종 중 하나에 배정** (플레이어가 선택 불가, 결과만 표시)
- 추가로 **플레이어 본인에게는 원하는 곳에 분배 가능한 스탯 +10** 별도 지급
- 토너먼트 결과에서 **16강·8강·4강·준우승·우승** 단계별로 명확히 표시

### 6.4 스킬 슬롯 규칙 (확정)
- 플레이어 스킬 슬롯 최대 **8개**
- 미달 시: 스킬 3개 후보 중 1개 선택하거나 **넘기기**
- 초과 시 (현재 8개): 기존 스킬 1개를 **교체** (삭제하고 새 스킬 획득)
- 넘기기를 선택하면 이번 회차 스킬 습득 없이 다음 단계로

## 11. 저장 데이터 전략 (확정)

### 11.1 저장 슬롯 시스템
- 플레이어당 **최대 3개 저장 슬롯**
- 각 슬롯은 **독립된 플레이 기록** (선택 캐릭터, 누적 스탯, 보유 스킬, 회차 번호)
- 예선·본선·토너먼트 어느 단계에서 탈락해도 **계속 성장 누적** (게임오버 없음)
- 슬롯이 모두 차 있고 새 게임을 시작하려면 → 덮어쓸 슬롯 1개를 선택하여 초기화

### 11.2 저장소 이원화

| 데이터 | 저장소 | 이유 |
|--------|--------|------|
| 로그인 인증 / 허용 이메일 확인 | Firebase Auth | 기존 플랫폼 공통 |
| 슬롯 프로필 (선택 캐릭터, 누적 스탯, 스킬, 회차) | **Firestore** | 기기 간 동기화 |
| 대회 **요약** (우승자, 회차 번호, 보상 내역) | **Firestore** | 장기 이력 |
| 전투 **상세 로그** (턴별 액션) | **IndexedDB** (브라우저 로컬) | 용량 대비 무료, 동기화 불필요 |
| 시뮬레이션 모드 대회 상세 | **IndexedDB** | 동일 |

### 11.3 Firestore 스키마 (요약)

```
/users/{userId}/slots/{slotId}  (slotId = 1 | 2 | 3)
  - characterId: string
  - cumulativeStats: { 체력, 힘, 민첩, 지력, 행운 }
  - skills: string[]           (보유 스킬 ID, 최대 8개)
  - currentRound: number
  - bestClearRound: number | null
  - createdAt, updatedAt: timestamp

/users/{userId}/slots/{slotId}/tournaments/{tournamentId}
  - round: number
  - winner: charId
  - stage: "완료"
  - playerResult: "우승" | "토너먼트" | "본선" | "예선" | "예선탈락"
  - rewards: { stat, amount, acquiredSkillId }
```

### 11.4 IndexedDB 스키마 (전투 로그)

```
DB: battleGrandPrix
  ObjectStore: matchLogs
    - matchId (key)
    - tournamentId, slotId, stage
    - log: [ { turn, actor, action, damage, ... } ]
    - createdAt
```

비용 관점: Firestore 무료 티어(1GB, 20K write/day) 범위 내. IndexedDB는 브라우저 로컬로 비용 0.
