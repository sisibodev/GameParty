## 2. 핵심 컨셉

### 시야 기반 스텔스
- 2D 탑다운 시점. 플레이어는 **자신의 시야 안쪽만 선명하게** 보이고, 시야 밖은 어두워짐 (fog of war).
- 벽/기둥 등 장애물에 의해 시야가 차단됨.
- 경찰은 도둑보다 **시야가 넓고 이동 속도가 빠름** → 기동성/수색 유리.
- 도둑은 **은신**과 **연막**으로 시야를 회피 가능 → 기습/도주 유리.

### 금고 해킹 — 숫자야구 미니게임
- 맵 곳곳에 금고가 숨겨져 있다. 일부는 **빈 금고**이며 열기 전에는 알 수 없다.
- 금고 비밀번호는 **숫자야구**로 맞춘다 (예: 3자리 또는 4자리).
- **시도 횟수 제한**이 있고, 초과 실패 시 **사이렌이 울려** 해당 위치가 경찰 미니맵에 표시된다.
- 맞추면 보물 획득. 보물은 **도둑 팀 공용 카운터**에 누적된다.

### 팀 승리 조건

| 팀 | 승리 조건 |
|----|-----------|
| 도둑 | **팀 목표 보물 개수**를 모은 뒤, **절반 이상**의 도둑이 탈출문으로 나감 |
| 경찰 | 모든 도둑을 체포 (또는 제한 시간 내 탈출 실패 유도) |

## 3. 게임 규칙

### 3.1 도둑

- **목표**: 금고를 털어 목표 보물 개수 달성 → 탈출문으로 나감.
- **이동**: 경찰보다 느림.
- **탈출**: 탈출문을 찾아야 하며, 위치는 맵마다 상이.
- **협동 요소**:
  - 잡힌 동료 도둑이 갇힌 **유치장**을 찾아가 문을 열어 구출 가능.
  - 단, 잡힌 직후 **3분간은 구출 불가** (강제 대기 시간).
  - 대기 시간은 도둑 팀 전체에 공통 표시.
- **승리**: 팀 목표 보물 확보 + 도둑 절반 이상 탈출.

#### 3.1.1 능력 (공통)

| 능력 | 효과 | 비고 |
|------|------|------|
| 은신 | 주변 물건으로 위장. 경찰이 지나쳐도 발각 안 됨 | **경찰이 때리면 풀림**, 쿨타임 있음 |
| 연막 | 범위 내 시야 차단 | 쿨타임 있음 |

#### 3.1.2 보물 획득 규칙

- 보물 카운터는 **도둑 팀 공용** (어느 도둑이 땄든 팀 합산).
- 보물 획득 후 체포되어도 **카운터는 감소하지 않음**.
- 즉 "땄으면 끝" — 경찰은 획득 자체를 막아야 함.

#### 3.1.3 금고 규칙 상세

- 금고는 맵 전역에 분산 배치, 일부는 **빈 금고**.
- 빈 금고인지 여부는 **열어보기 전에는 알 수 없음**.
- 한 번 연 금고는 **열린 상태로 유지** (다른 도둑도 빈 금고 여부 확인 가능 → 정보 공유).
- 숫자야구 시도 횟수를 **초과 실패**하면 경보 발동 → 미니맵에 위치 노출.

#### 3.1.4 체력 / 피격 규칙 — **✅ 확정 (v0.3)**

- 도둑은 **경찰에게 3번 맞으면 체포** (유치장 이송).
- **피격 스택은 라운드 내내 누적** — 회복 수단 없음 (긴장감 유지).
- **은신 중 피격**은 **은신 해제만 발동**, 피격 스택은 **증가하지 않음** (은신 가치 보호).
- 피격 시 짧은 경직 + 피격 이펙트.
- UI: 도둑 본인 화면 하단에 **피격 게이지 3칸** (1/2/3 단계 시각화).

### 3.2 경찰

- **목표**: 모든 도둑 체포 또는 제한 시간 내 탈출 저지.
- **이동**: 도둑보다 빠름.
- **시야**: 도둑보다 넓음.
- **미니맵**:
  - 금고 경보 발생 시 해당 위치 자동 표시.
  - 동료 경찰이 도둑을 **시야에 잡으면** 해당 도둑 위치 미니맵 공유.
  - 쫓던 경찰이 **시야에서 3초 이상 놓치면** 미니맵 표시 해제.
- **체포**: 도둑과 접촉 시 유치장으로 호송.

#### 3.2.1 탐지 능력 "스캔" — **✅ 확정 (v0.3)**

경찰이 도둑을 너무 못 찾는 상황을 보완하기 위한 능동적 탐지 스킬.

| 항목 | 값 |
|------|-----|
| 쿨타임 | **60초** |
| 지속 시간 | **2초** (발동 순간 위치 표시) |
| 탐지 반경 | 경찰 본인 중심 **7타일** |
| 표시 대상 | 경찰 본인 시야 + 미니맵 (팀 경찰 공유) |
| 예외 | **은신 중 도둑은 감지되지 않음** (은신 가치 유지) |
| UI | 경찰 하단 스킬 아이콘 (쿨타임 진행 표시) |

> 도둑 측에는 스캔 발동 피드백을 주지 않는다 (경찰 위치/행동을 역으로 읽히지 않도록).

### 3.3 방 생성 및 인원 비율

| 총 인원 | 경찰 | 도둑 | 비고 |
|---------|------|------|------|
| 4명 | 1 | 3 | 최소 |
| 5명 | 2 | 3 | |
| 6명 | 2 | 4 | |
| 7명 | 3 | 4 | |
| 8명 | 3 | 5 | |
| 10명 | 4 | 6 | 최대 |

- 목표 보물 개수는 **인원수에 비례**하여 자동 조정 (예: 도둑 1명당 2개).
- **봇은 경찰 포지션만** 채울 수 있음 (도둑 봇 없음).

### 3.4 시간 제한

- 라운드에 **시간 제한**(예: 15분) 존재.
- 시간 내 도둑이 탈출 조건을 달성하지 못하면 **도둑 패배**.

### 3.5 발소리 시스템 (공통) — **✅ 확정 (v0.3)**

경찰의 이동을 도둑이 **청각 단서**로 감지할 수 있게 하여, 정적인 탐색에 긴장감을 부여한다.

| 상태 | 발소리 반경 |
|------|-------------|
| 경찰 걷기 | **3타일** |
| 경찰 뛰기 | **6타일** |

- **도둑 측 UI**: 반경 내에 있는 도둑 화면에 **방향 인디케이터** (화살표 또는 원형 레이더)로 경찰의 대략적 방향 표시. 정확한 위치는 표시하지 않음.
- **경찰 측 UI**: 경찰 본인 화면에 **자기 발소리 범위**를 반투명 원으로 오버레이 → 스텔스 접근 가능 여부를 직관적으로 판단.
- **은신 중 도둑은 발소리를 듣지 못함? → 감지 여부에 영향 없음**: 도둑이 은신 중이어도 주변 경찰 발소리는 들림 (정보 비대칭 유지).
- **도둑 발소리는 발생하지 않음** (도둑은 상시 스텔스 포지션 — 은신 가치와 별개로 기본 무음).

### 3.6 탈출구 규칙 — **✅ 확정 (v0.3)**

탈출구가 경찰 수와 동일하면 경찰이 앞에 캠핑하여 승리가 가능해지므로, **개수 + 잠금 + 캠핑 비용** 3단 구조로 설계한다.

| 항목 | 값 |
|------|-----|
| 개수 | **경찰 수 + 1** (예: 경찰 3 → 탈출구 4). 맵 전역 분산 배치. |
| 초기 상태 | **잠김** — 도둑 팀이 **목표 보물 개수 달성 전**에는 사용 불가 |
| 해제 조건 | 도둑 팀 보물 카운터가 목표 달성 시 **모든 탈출구 동시 해제** |
| 해제 알림 | 양 팀 모두에게 "탈출구 개방" 전체 알림 |
| 캠핑 방지 | 탈출구 해제 후 경찰이 **탈출구 반경 영역에 15초 체류 시**, **다른 탈출구 위치**가 도둑 팀 전체 미니맵에 공개 |
| 리셋 | 경찰이 영역을 **모두 벗어나면** 공개 해제 (7.1과 동일 패턴) |
| 경찰 경고 UI | **"위치 노출 중"** 표시 (7.1과 동일한 UI 규격 재사용) |

> 탈출구 잠금은 "금고 → 탈출" 게임 플로우의 기본 순서를 강제한다. 해제 후 캠핑 비용은 유치장 캠핑 방지(7.1)와 동일한 **정보 비용** 원리.

## 4. 게임 흐름

```
[방 생성]
  └─ 방장이 인원/봇 설정
       └─ 경찰 vs 도둑 배정 (랜덤 또는 선택)

[게임 시작]
  └─ 맵 로드
       ├─ 도둑: 랜덤 스폰 지점
       └─ 경찰: 경찰서(중앙) 스폰

[진행 중]
  ├─ 도둑: 금고 탐색 → 숫자야구 → 보물 획득
  ├─ 경찰: 순찰 → 도둑 추적 → 체포
  └─ 유치장: 잡힌 도둑 대기 / 동료 구출 가능

[종료]
  ├─ 도둑 승리: 목표 보물 확보 + 절반 이상 탈출
  ├─ 경찰 승리: 모든 도둑 체포 또는 시간 만료
  └─ 결과 화면 → 랭킹 갱신
```

## 5. UI 구성

### 5.1 게임 화면 (공통)

- **메인 뷰**: 탑다운 2D 맵, 시야 기반 fog of war
- **상단 HUD**:
  - 남은 시간
  - 도둑 팀 보물 카운터 (N / 목표 N)
  - 도둑 탈출 수 (N / 도둑 총원)
- **하단 HUD**: 능력 쿨타임 (도둑만), 체포 도둑 목록 (경찰만)

### 5.2 도둑 전용

- **갇힌 동료 타이머**: 유치장에 잡힌 동료 각각의 3분 카운트다운
- **능력 아이콘**: 은신 / 연막 (쿨타임 표시)
- **금고 상호작용 UI**: 숫자야구 입력 패널 (시도 횟수 표시)
- **피격 스택 게이지**: 하단에 3칸 표시 (3.1.4 참조)
- **발소리 방향 인디케이터**: 주변 경찰 발소리 감지 시 방향 화살표/원형 레이더 (3.5 참조)

### 5.3 경찰 전용

- **미니맵**: 감지된 도둑 위치 마커 + 금고 경보 위치 표시
- **체포 알림**: 접촉 가능 범위 내 도둑 하이라이트
- **스캔 스킬 아이콘**: 쿨타임 진행 표시 (3.2.1 참조)
- **자기 발소리 범위 오버레이**: 현재 상태(걷기/뛰기)에 따른 반경 반투명 원 (3.5 참조)
- **"위치 노출 중" 경고**: 유치장/탈출구 캠핑으로 정보 유출 시 붉은 경고 UI (7.1 / 3.6 공통)

### 5.4 결과 화면

- 승패 결과
- 플레이어별 기여도 (보물 획득 수, 체포 수, 구출 수, 탈출 여부)
- 개인 전적 업데이트 내역

### 5.5 관전 모드 — **✅ 확정 (v0.3)**

잡힌 도둑의 대기 시간(3분)과 탈출 후 라운드 종료 대기 시간 동안의 시청 경험.

| 항목 | 내용 |
|------|------|
| 대상 | 잡힌 도둑 (유치장 대기 중) / 탈출 완료한 도둑 / 체포된 후 미구출 도둑 |
| 전환 키 | **Q / E** (또는 숫자 1~N)로 **도둑 팀원 시점** 순회 |
| 기본 시점 | 자기 화면 (유치장 / 탈출 대기 상태) |
| 경찰 시점 | **관전 불가** (정보 공정성 유지 — 도둑이 경찰 시야를 보면 정보 비대칭 붕괴) |
| UI | 화면 상단에 "관전 중: [팀원 이름]" 표시 + 해당 팀원 HUD 미러링 |

> 갇혀있는 동안의 무료함을 해소하고, 팀원과의 협동 감각을 유지시킨다.

## 6. 기술 설계 (초안)

### 6.1 렌더링

- **PixiJS**로 2D 탑다운 렌더링.
- 시야(fog of war): 마스크 레이어 또는 lighting 쉐이더 적용.
- 3D 전환은 향후 검토 (Three.js).

### 6.2 실시간 동기화 — **✅ 확정 (v0.3)**

**초기: Firebase Realtime Database** — 플랫폼 기본 스택 재사용, 서버 운영 부담 없음. 이동/시야/금고 상태 동기화에 활용.

**확장 이행 경로**: 인원 증가/레이턴시 문제/치팅 검증이 필요해지면 **Node.js + Socket.io** 도입으로 2단계 이행.

| 옵션 | 장점 | 단점 |
|------|------|------|
| Firebase Realtime DB (초기) | 서버 관리 불필요, 플랫폼 기본 스택 | 지연/대역폭 한계, 서버 로직 검증 약함 |
| Node.js + Socket.io (확장) | 저지연, 서버 권위 검증 가능 (치팅 방지) | 별도 서버 필요 (호스팅 비용/설정) |

### 6.3 Firestore 기록

- 유저별 경찰/도둑 승률
- 누적 보물 획득, 체포, 구출, 탈출 횟수
- 글로벌 랭킹 (시즌제 검토)

## 7. 밸런스 / 디자인 미결 사항

원본 메모에서 **고민** 또는 **미정**으로 표시된 항목 + 기획 중 도출된 결정 포인트.

### 7.1 경찰 유치장 캠핑 방지 — **✅ 확정안 (v0.3)**

도둑을 구출하러 오는 동료를 경찰이 유치장 앞에서 잠복하는 문제를, **경찰 행동 제한이 아닌 정보 비용** 방식으로 해결한다.

> **핵심 아이디어**: 경찰이 유치장 근처에 오래 머물면, 그 대가로 **각 도둑에게 본인 주변 금고 정보가 유출**된다. 경찰은 캠핑을 선택할 수 있지만 도둑 팀의 수색 효율이 올라가는 전략적 비용을 지불하게 된다.

#### 7.1.1 메커니즘 상세

| 항목 | 값 |
|------|-----|
| 발동 타이머 방식 | **영역 타이머** (경찰 개인 타이머 X, 해당 영역 체류 총량으로 누적) |
| 발동 임계값 | **15초** — 영역 내 경찰 체류 시간 합산 |
| 영역 정의 | 유치장 반경 (맵 설계 시 구체 수치 확정, 제안: 6~8 타일) |
| 공개 대상 | **도둑 팀 전체** (모든 도둑 각자가 정보 수령) |
| 공개 정보 | **각 도둑 본인 위치 주변의 가까운 금고 2~3개** (개인화 — 도둑마다 다른 금고 표시) |
| 열린 금고 구분 | 이미 열린 금고는 **별도 아이콘/색상**으로 구분 표시 |
| 빈 금고 구분 | **숨김** (열어보기 전에는 알 수 없다는 기존 규칙 유지) |
| 리셋 조건 | 경찰이 영역을 **모두 벗어나면** 정보 표시 숨김 복원 |
| 경찰 경고 UI | **"위치 노출 중"** 표시 (영역 진입 시점부터 노출 상태까지 시각화) |

> **v0.3 변경**: 공개 금고 기준이 "유치장 주변" → "각 도둑 본인 주변"으로 바뀜. 유치장 근처 금고는 경찰 지척이라 접근이 어려움 — 각 도둑 주변 금고로 바꿔 즉시 활용 가능한 정보로 만듦.

#### 7.1.2 UI 플로우

**경찰 측 UI 단계:**

```
[영역 진입] → "유치장 감시 중 N초" 카운트다운 표시
   └─ 15초 도달 → "위치 노출 중" 경고 UI (붉은 표시, 도둑 팀에 금고 위치가 새나감 알림)
        └─ 영역 이탈 → "노출 해제 중..." → 숨김 복원
```

- 카운트다운으로 플레이어에게 **선택권**을 준다 (남을지 이탈할지).
- 노출 중 상태에서도 경찰은 정상적으로 플레이 가능 (행동 제한 없음).

**도둑 측 UI:**

- 미니맵에 **본인 위치 주변 금고 2~3개** 마커 표시 (각 도둑마다 다른 금고, 열린 금고는 별도 색상).
- 표시 시점에 "경찰이 캠핑 중입니다 — 주변 금고 위치 공개" 알림 (1회 flash).
- 정보 출처가 "경찰 캠핑"임을 명확히 보여줌 (다른 경보와 혼동 방지).

#### 7.1.3 남은 세부 결정

이번 버전에서 확정하지 않고 프로토타입에서 조정:

- 영역 반경 실수치 (타일 몇 개)
- "가까운 2~3개"의 정확한 개수 — 맵 크기/금고 밀도에 따라 조정
- 교대 캠핑 처리 세부 (영역 타이머이므로 경찰 교대 무관 — 의도대로 작동)
- 리셋 시 즉시 숨김 vs 부드러운 페이드 (UX 결정)

#### 7.1.4 폐기된 대안

- ~~1안: 유치장 근처 경찰 이동 속도 감소 / 체포 불가~~ — 경찰 행동을 직접 제한하여 인위적
- ~~2안: 구출 시 즉시 연막 발동~~ — 캠핑 자체 억제 효과 약함 (구출 성공 시에만 발동)
- ~~3안: 유치장 다중 입구~~ — 맵 디자인 제약, 근본 해결 아님
- ~~4안: 감시 상한 경고 UI만 표시~~ — 게임 메커니즘 영향 없이 피드백만 제공, 약함

### 7.2 2D vs 3D — **✅ 확정 (v0.3)**

- **2D (PixiJS) 확정**. 3D 전환은 장기 옵션 (자산 제작 비용 부담).

### 7.3 서버 방식 — **✅ 확정 (v0.3)**

- **Firebase Realtime Database 우선** → 병목/치팅 검증 필요 시 **Socket.io 이행**.
- 상세 근거는 섹션 6.2 참조.

### 7.4 맵 수 / 크기 — **✅ 확정 (v0.3)**

- **초기 출시 맵 1개**, **대형 맵** (분산 수색/캠핑 억제 공간 확보).
- 실제 타일 수/금고 수/탈출구 수 등 수치는 **프로토타입(M1~M2)에서 튜닝**.

### 7.5 관전 모드 — **✅ 확정 (v0.3)**

- 섹션 5.5로 이동. **Q/E 키로 도둑 팀원 시점 순회**, 경찰 시점 관전 불가.

### 7.6 타격/체포 판정 — **✅ 확정 (v0.3)**

- **3회 피격 누적 → 체포** (섹션 3.1.4 참조).
- 은신 상태 피격은 은신 해제만 발동 (스택 +0).
- 판정 반경(근접/방향) 수치는 프로토타입에서 튜닝.

### 7.7 남은 튜닝 파라미터 (프로토타입에서 조정)

- 유치장 캠핑 영역 반경 / 탈출구 캠핑 영역 반경
- 스캔 탐지 반경 및 지속/쿨타임 미세 조정
- 발소리 반경 (걷기 3 / 뛰기 6 타일 — 실제 맵 스케일과 대조)
- 피격 판정 반경 및 경직 시간
- 맵 타일 수 / 금고 총 개수 / 목표 보물 비율
