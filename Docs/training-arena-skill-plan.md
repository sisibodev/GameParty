## 1. 개요

### 1.1 목적
- 캐릭터 전투의 **변수**를 만드는 핵심 요소
- 같은 스탯이어도 스킬 구성·강화·패시브에 따라 승패가 갈리도록 설계
- 플레이어의 장기 성장(매 전투 승리 시 스킬 학습, 매 라운드 패시브 습득)을 뒷받침

### 1.2 기본 규칙 (v0.5.0)
- **아키타입별 고정 스킬 풀**: 8개 아키타입 × 7~9개씩 총 **64종**
- **캐릭터 고정 3개 스킬**: 캐릭터마다 컨셉·성격에 맞게 설계자가 배정 (랜덤 아님)
- 플레이어 시작 시 선택한 캐릭터의 고정 3개 스킬 보유
- 전투 승리 시 상대 스킬 중 **1개를 선택 학습** → 누적 (최대 5개 슬롯)
- **스킬 강화 시스템**: 골드 소비로 각 스킬 최대 **10레벨**까지 강화

### 1.3 스킬 발동 조건
매 턴 행동 시점에 다음 순서로 체크:

```
1. 마나 충분한가? → No면 일반 공격
2. 쿨타임 완료? → No면 일반 공격
3. 적극성 체크 (확률 판정) → 실패 시 일반 공격
4. 사용 가능한 스킬 중 가중치 랜덤 선택 → 발동
```

## 2. 스킬 속성

### 2.1 공통 필드

| 필드 | 설명 |
|------|------|
| id | 고유 ID (예: `war_01`, `mag_02`) |
| name | 스킬명 |
| tier | 등급 (common / rare / hero / legend) |
| category | 분류 (attack / defense / buff / debuff / heal / special) |
| target | 대상 (self / single / all_enemies) |
| cost | 마나 소모량 |
| cooldown | 쿨타임 (턴 수) |
| priority | 발동 우선순위 (낮을수록 먼저) |
| description | 플레이어에게 보여주는 설명 |

### 2.2 등급별 기대 성능

| 등급 | 기대 효과 |
|------|-----------|
| common | 단일 타겟, 낮은 배율, 짧은 쿨타임 |
| rare | 부가 효과 포함 (상태이상, 조건부 효과) |
| hero | 강력 단일 또는 광역 + 조건부 버프 |
| legend | 판 뒤집기급 효과, 긴 쿨타임 |

## 5. 패시브 스킬 시스템 (v0.5.0)

### 5.1 개요
- **총 50개** 패시브 스킬 (data/passiveSkills.json)
- 캐릭터당 **최대 6개** 보유 (MAX_PASSIVE_SLOTS = 6)
- 패시브는 등급 구분 없음 (모두 동일 등급)

### 5.2 패시브 습득 방법

#### A. 전투 승리 후 (상대 패시브 선택)
- 전투 승리 → 스킬 선택 완료 → **상대가 보유한 패시브 전부 표시**
- 플레이어가 이미 보유한 패시브는 비활성(흐리게) 표시
- 1개 선택 또는 건너뛰기
- 최대 6개 초과 불가

#### B. 라운드 종료 후 (랜덤 패시브 보상)
- 라운드 종료 → PassiveRewardPage 진입
- 미보유 패시브 중 **3개 랜덤 제시**
- 1개 선택 또는 건너뛰기 → skill_select 페이즈로 이동

## 6. 전투 수치 공식 (스킬 연동)

### 6.1 데미지 공식
```
기본 데미지 = ATK × 배율
강화 보정   = 기본 데미지 × 강화 배율  (SKILL_ENHANCE_MULT[level-1], level=0이면 ×1.0)
방어 공식   = 강화 보정 데미지 × DEF_FORMULA_BASE / (DEF_FORMULA_BASE + DEF × (1 - def_break_pct))
크리 보정   = 발동 시 × (1.5 + luk×0.001 + crit_damage_bonus)
최소 데미지 = 1
```

### 6.2 치유 공식
```
회복량 = HP성장 × 배율
- 최대 HP 초과분은 버림
```

### 6.3 지속 효과 (DoT / HoT)
```
턴 시작 시점 처리:
  - regen 패시브: maxHp × pct% 회복
  - berserker_rage: HP 조건 충족 시 ATK 임시 증가
  - last_stand_spd: HP 조건 충족 시 SPD 임시 증가
피격 후 처리:
  - lifesteal: 가한 피해 × pct% 회복
  - thorns: 받은 피해 × pct% 반사 (최소 1)
```

### 6.4 적극성 → 스킬 발동 확률
```
기본 발동률 = 50% + (적극성 × 0.5%)
- 적극성 100 → 100% 발동 시도
- 마나/쿨타임 모두 충족 + 확률 판정 통과 시 스킬 선택
```
