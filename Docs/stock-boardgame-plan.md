## 2. 핵심 컨셉

### 숨겨진 시나리오 방식
- 게임 시작 시, 각 회사의 **전체 라운드 가격 변동이 미리 결정**된다.
- 플레이어들은 이 흐름을 **알 수 없는 상태**에서 매매를 진행한다.
- 매 라운드 종료 시 차트가 공개되며, 다음 흐름을 예측해야 한다.

### 카드 심리전
- **특수 카드**: 특정 회사의 수익률을 몰래 조작. 누가 어디에 썼는지 비공개.
- **정보 카드**: 회사 정보, 타 플레이어 동향, 라운드 이벤트 등을 엿볼 수 있는 첩보 카드.
- **라운드 카드**: 매 라운드 시작 시 미리 추첨되며, 결과 발표 때 공개. 전체 회사에 영향.

### 순위 긴장감 유지
- 게임 중 **등수만 공개**, 자산 총액은 비공개 (자신의 금액은 자기 화면에만 표시)
- 순위표는 **Tab 키를 누르는 동안만 표시** (홀드형, 떼면 사라짐)
- 1등과의 격차를 알 수 없어 마지막까지 긴장감 유지

## 3. 게임 규칙

### 3.1 기본 구조

| 항목 | 기본값 | 방 생성 시 변경 가능 여부 |
|------|--------|--------------------------|
| 시작 자금 | 100만원 | 검토 중 |
| 회사 수 | 인원수 기반 자동 | ✅ 자동 or 방장 수동 선택 |
| 라운드 수 | 10라운드 | ✅ 가능 |
| 라운드 제한 시간 | 3분 | ✅ 가능 |

- **승리 조건**: 최종 라운드 종료 후 보유 현금 + 주식 평가액 합계 1위

### 3.2 회사 수 설계

- **최소 4개** / 플레이어 수보다 적게 유지
- 방 생성 시 두 가지 모드 선택:

| 모드 | 설명 |
|------|------|
| 자동 (유동) | 인원수에 따라 자동 결정 |
| 수동 | 방장이 직접 선택 (최소 4개 ~ 인원수-1개) |

**자동 모드 기준**

| 인원 수 | 회사 수 |
|---------|---------|
| 2~3명 | 4개 |
| 4~5명 | 5개 |
| 6~7명 | 6개 |
| 8명 이상 | 7개 |

### 3.3 상장폐지 및 신규 상장

- 주식 최저 가격은 **100원**
- 100원에 도달하면 **상장폐지** — 해당 회사 소멸, 투자된 금액 전액 손실

**상장폐지 연출 순서**

```
① 라운드 결과창
     - 해당 회사 카드에 "상장폐지" 마크 표시
     - 투자자에게 손실 금액 별도 안내

② 다음 라운드 시작 시
     - 종목 패널에서 해당 회사 칸이 부서지는 연출
     - 잔해 속에서 새로운 회사명 등장 애니메이션

③ 신규 상장 회사
     - 게임 시작 시 회사 생성 방식과 동일하게 랜덤 생성
       (회사명, 유형, 시작가 모두 랜덤)
     - 신규 상장가로 즉시 거래 가능
```

### 3.4 파산 구제 시스템

- **주식 보유량이 0**이고, 보유 현금으로 **어떤 주식도 1주조차 살 수 없는 상황**이 되면 자동 발동
  - 주식을 1주라도 보유 중이면 구제 미발동 (주식을 팔아 현금을 마련할 수 있으므로)
- **100만원 리필** 지급
- 최종 결과에서 **리필된 금액만큼 자산에서 차감** 후 순위 산정

### 3.5 현금 보유세

- 매 라운드 종료 시 보유 현금에서 세금 자동 차감 (투자 유인 강화)
- **주식 평가액은 과세 대상에서 제외** — 보유 현금(cash) 금액에만 세율 적용
- 라운드가 뒤로 갈수록 세율 상승
- 라운드 결과 화면 정산 단계에서 명시적으로 표시

**세율 구간**

| 라운드 | 세율 |
|--------|------|
| 1~2라운드 | 1% |
| 3~4라운드 | 2% |
| 5~6라운드 | 3% |
| 7~8라운드 | 4% |
| 9~10라운드 | 5% |

- Tab 홀드 오버레이에서 전체 라운드 세율 구간 확인 가능
- 상단 HUD에 이번 라운드 세율 + 다음 라운드 세율 상시 표시

### 3.6 라운드 진행 순서

```
1. [카드 드래프트]    매매 시간 내 포함 — 플레이어 수 +1장 공개
                     꼴지부터 순서대로 1장씩 선택 (선택한 카드 하단에 선택자 이름 표시)
                     모든 플레이어 선택 완료 후 카드 손패에 추가, 남은 1장 소멸 (2라운드부터)
2. [매매 페이즈]      제한 시간 — 매매(% 단위 포함) + 정보 카드(라운드당 1장) + 특수 카드 지정
3. [레디]             모든 플레이어가 레디 버튼을 누르면 즉시 라운드 종료
                      (시간이 남아있어도 전원 레디 시 자동으로 결과 화면으로 이동)
4. [카드 연출]        특수 카드 순차 공개 애니메이션 (회사별 수익률 실시간 변동)
5. [라운드 카드]      결과 화면에서 이번 라운드 이벤트 카드 공개
6. [최종 수익률]      모든 효과 반영 후 최종 등락률 + 금액 + 수익률(%) 확정 표시
7. [상장폐지 처리]    100원 이하 도달 시 상장폐지 연출 + 신규 회사 등장 (해당 시)
8. [정산]             현금 보유세 차감 + 현금 소각 적용(해당자) + 파산 구제 발동(해당자) 표시
9. [순위 공개]        등수만 표시, 자신의 총 자산은 본인에게만 표시
10. [보너스 카드]     3라운드마다 등수 기반 추가 카드 + 특수카드 무효 지급
11. [다음 라운드]     반복
```

### 3.7 매매 규칙

| 항목 | 내용 |
|------|------|
| 매수 | 현재 가격 × 수량만큼 현금 차감, 주식 추가 |
| 매도 | 보유 수량 이하로만 가능, 현금 증가 |
| 공매도 | 미지원 (v1) |
| 신용거래 | 미지원 (v1) |
| 매매 단위 | 1주 단위 |
| 제한 시간 | 설정값 (기본 3분) |

**수량 조절 방식 (구현 완료)**
- `-10` / `-1` 버튼, 직접 숫자 입력, `+1` / `+10` 버튼
- **최대 매수**: 현재 현금으로 살 수 있는 최대 수량 자동 세팅
- **전부 매도**: 보유 수량 전체를 자동 세팅
- **% 단위 매수**: 현재 보유 현금 기준 25% / 50% / 75% 버튼으로 빠른 매수
- **% 단위 매도**: 현재 보유 수량 기준 25% / 50% / 75% 버튼으로 빠른 매도

### 3.8 주가 변동 설계

- 게임 시작 시 랜덤 시드로 전체 라운드 가격 곡선 사전 생성
- 지나간 라운드의 가격만 차트로 확인 가능, 이후는 숨겨져 있음

**변동폭 설계**

| 구분 | 변동폭 | 등장 빈도 |
|------|--------|----------|
| 일반 구간 | ±5~15% | 대부분의 라운드 |
| 도파민 구간 | ±30~50% 이상 | 가끔 랜덤 등장 |

**회사 유형**

| 유형 | 설명 | 도파민 빈도 |
|------|------|------------|
| 안정형 | 변동성 낮고 완만한 성장 | 낮음 |
| 성장형 | 초반 낮고 후반 급등 가능 | 중간 |
| 하락형 | 전반적으로 우하향 트렌드 | 낮음 |
| 롤러코스터형 | 큰 폭의 상하 반복 | 높음 |
| 반전형 | 초반 하락 후 후반 급반등 | 중간 |

## 4. 카드 시스템

### 4.1 카드 종류 분류

| 분류 | 설명 | 사용 시점 | 라운드당 제한 |
|------|------|----------|--------------|
| 특수 카드 | 특정 회사의 수익률 조작 (비공개) | 매매 페이즈 중 | 기본 2장 |
| 정보 카드 | 회사/플레이어/이벤트 정보 열람 | 매매 페이즈 중 언제든 | 1장 |
| 라운드 카드 | 전체 회사 일괄 이벤트 (자동 등장) | 라운드 결과 시 자동 | — |

---

### 4.2 특수 카드

#### 게임 시작 시 기본 지급 (전원 동일, 8장)

| 카드명 | 효과 | 수량 |
|--------|------|------|
| 급등 | 해당 회사 이번 라운드 수익률 +20%p | 3장 |
| 급락 | 해당 회사 이번 라운드 수익률 -20%p | 2장 |
| 증폭 | 해당 회사 이번 라운드 수익률 배율 증폭 (중첩 체감: 1장 ×2.0, 2장 ×3.5, 3장 ×4.7) | 1장 |
| 감쇠 | 해당 회사 이번 라운드 수익률 배율 감쇠 (중첩 체감: 1장 ×0.50, 2장 ×0.33, 3장 ×0.24) | 1장 |
| 반전 | 해당 회사 이번 라운드 수익률 부호 반전 (+ ↔ −) | 1장 |
| **합계** | | **8장** |

#### 사용 규칙
- 매 라운드 기본 **2장** 사용 가능 (특급 카드로 3장까지 확장 가능)
- **1번째 카드**: 어느 회사에나 자유롭게 사용
- **2번째 카드**: 반드시 1번째와 **다른 회사**에만 사용
- **3번째 카드** (특급 카드 사용 시): 앞서 쓴 회사와 반드시 **다른 회사**에만 사용
- 즉 한 라운드에 **같은 회사에 본인 카드 2장 이상 집중 불가**
- 사용 후 소멸, 누가 어느 회사에 썼는지 **비공개**
- 다른 플레이어의 카드는 같은 회사에 중첩 적용 가능
- **사용 중인 카드 현황은 매매 페이즈 내 별도 패널에 상시 표시** (까먹음 방지)

#### 증폭 / 감쇠 중첩 배율표

중첩 시 추가 효과가 체감 감소하여 무한 중첩 남용을 방지한다.

| 중첩 장수 | 증폭 배율 | 감쇠 배율 |
|----------|----------|----------|
| 1장 | ×2.0 | ×0.50 |
| 2장 | ×3.5 | ×0.33 |
| 3장 | ×4.7 | ×0.24 |
| 4장 | ×5.7 | ×0.19 |

- 증폭/감쇠는 수익률 부호에 관계없이 배율 적용 (음수 수익에 증폭 → 손실 확대 리스크 존재)
- 주가가 100원 이하로 내려가면 상장폐지 발동 (감쇠 중첩으로도 극단적 하락 시 상장폐지 가능)

기본 지급 카드 외에 드래프트와 보너스 카드 풀에는 아래 카드도 포함:

| 카드명 | 효과 | 대상 | 비고 |
|--------|------|------|------|
| 소급등 | 해당 회사 수익률 +10%p | 회사 지정 | — |
| 소급락 | 해당 회사 수익률 -10%p | 회사 지정 | — |
| 급등 | +20%p | 회사 지정 | — |
| 급락 | -20%p | 회사 지정 | — |
| 증폭 | 배율 증폭 (중첩 체감) | 회사 지정 | — |
| 감쇠 | 배율 감쇠 (중첩 체감) | 회사 지정 | — |
| 반전 | 부호 반전 | 회사 지정 | — |
| 결과 교체 | A회사와 B회사의 이번 라운드 최종 수익률을 서로 교환 | 회사 2개 지정 | — |
| 특수카드 무효 | 특정 회사에 쌓인 모든 특수 카드 효과 무효화 (라운드 카드 및 순수 등락 유지) | 회사 지정 | 3라운드 보너스 최하위 지급 + 드래프트 확률 등장 |
| 포트폴리오 저격 | 대상 플레이어의 비중 1위 종목에 급락 자동 적용 | 플레이어 지정 | — |
| 수익 강탈 | 대상 플레이어의 이번 라운드 수익 30%를 내가 흡수 | 플레이어 지정 | — |
| 집중 저격 | 미행 카드로 확인한 종목 중 직접 지정, 감쇠보다 큰 폭으로 하락 | 플레이어 + 종목 지정 | 미행 카드 선사용 필요 |
| 현금 소각 | 대상 플레이어 보유 현금 50% 소멸 (라운드 결과 정산 시 차감, 적용 중 본인 화면에 표시) | 플레이어 지정 | — |
| 집중 투자 강제 | 대상 플레이어가 현금의 75% 이상을 한 종목에 매수해야 함. 미이행 시 자동 집행: 비중 낮은 종목부터 순차 매도 → 비중 가장 높은 종목으로 집중 매수하여 75% 조건 충족 | 플레이어 지정 | — |
| 거래 정지 | 대상 플레이어가 이번 라운드 지정 종목 매매 불가 | 플레이어 + 종목 지정 | — |
| 손바꿈 | 대상 플레이어와 보유 현금 전액 교환 | 플레이어 지정 | 드래프트 낮은 확률 등장 |
| 라운드 카드 선택권 | 10라운드 적용 라운드 카드를 목록에서 직접 선택 | 없음 (본인) | 9라운드 꼴등에게만 지급 |

### 4.3 정보 카드

- 매매 페이즈 중 **언제든지** 사용 가능
- **라운드당 1장 제한** (특급 카드 사용 시 2장으로 확장)
- 사용 즉시 효과 발동 후 소멸
- 결과는 **사용한 본인에게만** 표시
- 스캔 계열 카드는 **사용한 턴 내내 변동 상황을 실시간 반영** — 닫을 수 있는 팝업이 아닌 **매매 화면 하단에 고정된 스캔 정보 패널**로 표시되며, 라운드가 끝날 때까지 유지됨. 여러 장 사용 시 행이 추가됨

```
┌──────────────────────────────────────────┐
│  📡 스캔 정보 (실시간)                    │
│  등락 예보 — A사 ▲ / C사 ▼              │
│  카드 탐지 — B사 현재 3장 쌓임           │
│  미행    — 홍길동: A사, C사 투자 중      │
└──────────────────────────────────────────┘
```

| 카드명 | 효과 | 구현 |
|--------|------|------|
| 등락 예보 | 내가 선택한 회사의 이번 라운드 순수 등락 방향 공개 (▲/▼, 특수카드 효과 제외) — 턴 내 실시간 유지 | ✅ 구현 완료 |
| 수익률 투시 | 내가 선택한 회사의 이번 라운드 순수 수익률 수치까지 공개 (정확한 % 수치, 특수카드 효과 제외) — 드래프트 낮은 확률 등장 | ✅ 구현 완료 |
| 카드 탐지 | 내가 선택한 회사에 현재 사용된 특수 카드 총 장수 공개 — 턴 내 실시간 업데이트 | ✅ 구현 완료 |
| 카드 감정 | 내가 선택한 회사에 적용된 특수 카드 종류 목록 공개 (장수가 아닌 어떤 카드인지까지) — 드래프트 낮은 확률 등장 | ✅ 구현 완료 |
| 미행 | 내가 선택한 플레이어가 이번 라운드 투자한 회사명 목록 공개 (금액 비공개) — 턴 내 실시간 업데이트, 집중 저격 카드와 연계 가능 | ✅ 구현 완료 |
| 내부고발 | 이번 라운드 특수 카드가 가장 많이 쌓인 회사 1곳 공개 | ✅ 구현 완료 |
| 시장 스캔 | 전체 회사 중 이번 라운드 순수 등락 상위 2개 / 하위 2개 회사명만 공개 — 턴 내 실시간 유지 | ✅ 구현 완료 |
| 포트폴리오 스캔 | 내가 선택한 플레이어의 현재 보유 종목 목록 공개 (수량/금액 비공개) | ✅ 구현 완료 |
| 특급 카드 | 이번 라운드 특수 카드를 1장 추가 사용 가능 + 정보 카드도 1장 추가 사용 가능 | ✅ 구현 완료 |
| 이벤트 예보 | 이번 라운드 이벤트 카드를 미리 확인 (이름 + 효과 표시) | ✅ 구현 완료 |

> 정보 카드는 드래프트 및 3라운드 보너스 카드 풀에 포함되어 랜덤 획득 가능
> 기본 지급 카드에는 포함되지 않음

---

### 4.4 카드 드래프트 (2라운드부터 매 라운드)

- 제한 시간 **내에 포함** (별도 시간 없음)
- **플레이어 수 + 1장** 공개 (예: 6명이면 7장 공개)
- **꼴지부터 순서대로** 1장씩 선택, 1인당 선택 시간 **20초 제한**
- 20초 내 미선택 시 남은 카드 중 **랜덤으로 자동 지급**
- 선택된 카드 하단에 **선택자 이름 표시** (누가 뭘 가져갔는지 모두에게 공개)
- 모든 플레이어 선택 완료 후 카드가 각자 손패에 추가
- 남은 1장은 소멸
- 드래프트 풀: 특수 카드 + 정보 카드 혼합 (손바꿈은 낮은 확률, 특수카드 무효는 일반 확률)

---

### 4.5 3라운드마다 보너스 카드 지급 ✅ 구현 완료

- 3, 6, 9라운드 결과 발표 후 (= 4, 7, 10라운드 시작 시) 등수 기반 차등 지급
- **2명씩 묶어서** 등수별 차등 지급

| 등수 묶음 | 일반 카드 | 추가 지급 |
|-----------|----------|----------|
| 상위 2명 | 1장 | — |
| 중위 2명 | 2장 | — |
| 최하위 나머지 | 3장 | 특수카드 무효 1장 |

- 인원 수에 따라 자동 조정 (예: 5명 → 상위 2명 1장, 중위 2명 2장, 5등 3장 + 특수카드 무효)
- 지급 카드는 특수 카드 + 정보 카드 풀에서 **랜덤** 지급
- **9라운드 꼴등**에게는 추가로 **라운드 카드 선택권** 1장 지급 (10라운드 라운드 카드를 목록에서 직접 선택)

### 4.6 라운드 카드 ✅ 구현 완료

- 매 라운드 **시작 시 미리 추첨**하여 RTDB에 저장 (결과 발표 전까지 비공개)
- 이벤트 예보 카드를 사용한 플레이어만 미리 확인 가능
- **전체 회사**에 동시 적용
- 특수 카드 공개 연출 완료 후 마지막에 공개
- **균등 확률**로 랜덤 등장 (총 14장 풀)

| 카드명 | 효과 |
|--------|------|
| 시장 활황 | 전 종목 수익률 +10%p |
| 시장 폭락 | 전 종목 수익률 -10%p |
| 대호재 | 전 종목 수익률 +25%p |
| 대폭락 | 전 종목 수익률 -25%p |
| 변동성 확대 | 전 종목 수익률 절댓값 ×1.5 (방향 유지) |
| 변동성 축소 | 전 종목 수익률 절댓값 ×0.5 (방향 유지) |
| 역전의 날 | 전 종목 수익률 부호 반전 (+ ↔ −) |
| 무풍지대 | 전 종목 수익률 0% 고정 (특수 카드 효과도 무효) |
| 양극화 | 등락 상위 절반 +15%p, 하위 절반 -15%p |
| 버블 | 상승 종목 수익률 ×1.3 |
| 공황 | 전 종목 수익률 ×0.7 |
| 금리 인상 | 전 종목 수익률 -5%p |
| 유동성 장세 | 전 종목 수익률 +5%p |
| 뉴스 블랙아웃 | 이번 라운드 정보 카드 무효 |

## 5. 라운드 결과 연출 ✅ 구현 완료

### 연출 순서

```
① 기본 주가 변동 표시 (특수/라운드 카드 효과 제외한 순수 등락)
     - 1.2초간 표시

② 회사별 특수 카드 공개 애니메이션
     - 카드 뒷면 → 앞면 플립 (주식 캔들스틱 디자인)
     - 카드 공개 시마다 해당 회사의 수익률 실시간 변동 표시
     - 카드 1장당 약 1.3초 간격
     - 누가 사용한 카드인지는 비공개 (익명 처리)

③ 라운드 카드 공개
     - 특수 카드 전부 공개 후 등장 (큰 카드 플립 연출)
     - 전체 회사 수익률 최종값으로 일괄 업데이트

④ 상장폐지 처리 (해당 시)
     - 100원 이하 도달 회사 카드에 "상장폐지" 마크 표시
     - 투자자 손실 금액 안내
     - 다음 라운드 시작 시 해당 칸 부서지는 연출 + 신규 회사 등장

⑤ 정산
     - 현금 보유세 차감 표시 (라운드별 세율 명시)
     - 현금 소각 적용 대상자 차감 표시
     - 파산 구제 발동 대상자 100만원 리필 표시

⑥ 결과 확정 후 표시
     - 회사별 스파크라인 차트 (라운드 0 ~ 현재 가격 추이)
     - 내 포트폴리오 수익표: 보유 종목별 등락% + 이번 라운드 수익금 + 수익률(%)
     - 총 자산: 현금 + 주식 평가액 합계 (본인에게만)
     - 순위: 등수만 표시, 내 행에만 총 자산 금액 추가 표시
```

**정산 표시 예시**
```
[내 수익 정산]
  주식 평가 수익       +₩ 320,000  (+32.0%)
  현금 보유세 (3%)    - ₩  37,200
  현금 소각 (50%)     - ₩ 150,000   ← 해당자만 표시
  ─────────────────────────────────
  이번 라운드 최종     +₩ 132,800  (+13.3%)
```

### 카드 디자인
- 특수 카드 뒷면: 짙은 남색 배경, 캔들스틱 SVG 패턴, 📈 아이콘, 모서리 ?
- 라운드 카드 뒷면: 보라색 계열, 🌐 아이콘
- 카드 앞면: 카드 타입명 + 효과 설명 + 대상 회사
- **결과 화면에서 특수 카드 사용자 이름은 표시하지 않음** (익명 처리 — 심리전 유지)

### 6.1 로비 홈 (방 만들기 / 방 입장)

```
┌─────────────────────────────────────────┐
│          ← 뒤로가기 (좌상단)             │
│                                         │
│        📈 주식 보드게임                  │
│                                         │
│  ┌──────────────────────────────────┐   │
│  │  [방 만들기]    [방 입장]  탭     │   │

│  │  닉네임 입력                     │   │
│  │  라운드 수 선택  (드롭다운)       │   │
│  │  제한 시간 선택 (드롭다운)        │   │
│  │  회사 수 모드   (자동/수동)       │   │
│  │                                  │   │
│  │       [ 방 만들기 ]              │   │
│  └──────────────────────────────────┘   │
│                                         │

```

- **방 만들기 탭**: 닉네임 + 게임 설정 후 방 생성 → 6자리 코드 발급
- **방 입장 탭**: 닉네임 + 6자리 코드 입력 → 대기실 입장
- 카드: 다크 그라디언트 + 20px 라운드 + 퍼플 네뷸라 배경
- 버튼: 퍼플 그라디언트 + 퍼플 글로우

### 6.2 게임 대기실

```
┌─────────────────────────────────────────┐
│  주식 보드게임          방 코드: AB12CD  │
│                                         │
│  ┌─── 게임 설정 ───────────────────┐    │
│  │  라운드 수: 10  /  제한 시간: 3분 │    │
│  │  회사 수: 자동 (6명 기준 6개)    │    │
│  └──────────────────────────────────┘   │
│                                         │
│  플레이어 (4/8)                          │
│  ┌────────────────────────────────────┐ │
│  │  👑 홍길동   ● 준비완료 (초록 배지) │ │
│  │     김철수   ○ 대기중  (회색 배지)  │ │
│  │     나       ○ 대기중              │ │
│  │     이영희   ● 준비완료            │ │
│  └────────────────────────────────────┘ │
│                                         │
│  [  준비완료  ]   ← 토글 (퍼플/그린)    │
│  [게임 시작]  ← 방장에게만, 전원 준비 시 │
└─────────────────────────────────────────┘
```

- **준비 버튼**: 본인용 토글 (대기중 = 퍼플 / 준비완료 = 그린 글로우)
- **게임 시작 버튼**: 방장에게만 노출, 전원 준비 완료 시 활성화
- **설정 패널**: 방장만 수정 가능, 설정 변경 시 전원 준비 자동 해제
- 플레이어 준비 상태 실시간 반영

### 6.3 메인 게임 화면 (매매 페이즈) ✅ 구현 완료

```
┌───────────────────────────────────────────────────────────────────┐
│  [6라운드]  ████████ 타이머 ████████  세율: 3%→4%  [₩850,000]  [?][해산] │
├──────────────────────┬────────────────────────────────────────────┤
│  종목 패널 (좌)       │  차트 영역 (선택 종목 가격 차트)            │
│                      │                                            │
│  [🏭 A사  ₩12,400   │                                            │
│   보유 5주  +8.3%]   ├────────────────────────────────────────────┤
│  [🔬 B사  ₩8,100    │  매매 영역                                  │

│  ...                 │  [-10][-1][ 수량 입력 ][+1][+10]            │
│  ─────────────────  │  [25%][50%][75%] 매수  [25%][50%][75%] 매도 │
│  상대방 패널 (하단)   │  [최대 매수 ↑]        [전부 매도 ↓]         │

│  김철수 고민중 🟡    ├────────────────────────────────────────────┤
│  이영희 레디완료 🟢  │  드래프트: [카드A-홍길동] [카드B] [카드C]   │
│                      │  특수 카드: [급등] [증폭] ...               │
│                      │  정보 카드: [등락예보] ...                  │

└────────────────────────────────────────────────────────────────────┘
```

- **상대방 패널**: 이름 + 상태(거래중 🔴 / 고민중 🟡 / 레디완료 🟢)만 표시 — 자산 비공개
- **레디 버튼**: 화면 하단 중앙 고정 (frosted glass 스타일). 토글 가능. 전원 레디 시 자동 라운드 종료
- **드래프트 패널**: 매매 영역에 인라인 표시 — 라운드 카드는 결과 화면에서만 공개
- **카드 사용 현황 패널**: 이번 라운드 사용한 카드와 대상 회사 상시 표시
- **Tab 홀드**: 순위 + 회사 가격 추이 + 세율 구간 오버레이
- **(?) 버튼**: 회사 유형 / 특수 카드 목록 / 라운드 카드 목록 3탭 모달

```
└──────────────────────────────────────────────────┘
```

### 6.4 라운드 결과 화면 ✅ 구현 완료

**3컬럼 레이아웃 (3 : 4 : 3 비율)**

```
┌──────────────────┬─────────────────────────┬──────────────────┐
│  가격 추이 (3)   │   주가 변동 + 연출 (4)  │  내 수익표 (3)  │
│                  │                         │                  │
│ 회사별 스파크라인 │ 회사명 · 수익률 · 최종가 │ 종목별 보유/등락 │
│ 전폭 그래프      │ (중앙 카드 형태 표시)   │ 이번 라운드 수익 │
│ (전체 라운드)    │                         │ 금액 + 수익률(%) │
│                  │ 특수 카드 플립 연출      │ 현금 + 주식 평가 │
│                  │ (익명 — 사용자 비공개)  │ = 총 자산        │
│                  │ 라운드 이벤트 카드       │                  │
│                  │ 상장폐지 연출 (해당 시)  │ 현금 보유세 차감 │
│                  │ 정산 표시               │ 현금 소각 (해당) │
│                  │ (세금/소각/구제)        │                  │
│                  │ 현재 순위 (등수 표시)   │                  │
│                  │ [다음 라운드] 버튼       │                  │
└──────────────────┴─────────────────────────┴──────────────────┘
```

- 특수 카드 플립 연출: 뒷면 → 앞면, 사용자 이름 **비공개**
- 순위 패널: 등수만 표시 (자산 총액 비공개), 본인 행에만 총 자산 금액 추가

### 6.5 최종 결과 화면 ✅ 구현 완료

```
┌─────────────────────────────────────────┐
│  🏆 최종 결과                            │
│                                         │
│  ┌─ 우승자 배너 ──────────────────────┐ │
│  │  🥇  홍길동   ₩2,340,000          │ │
│  └─────────────────────────────────────┘ │
│                                         │
│  순위  플레이어   최종 자산  라운드 수익 │
│  1위   홍길동   ₩2,340,000   +134%     │
│  2위   김철수   ₩1,870,000    +87%     │
│  3위 ★ 나       ₩1,520,000    +52%     │
│  4위   이영희   ₩1,200,000    +20%     │
│                                         │
│  [ 순위 추이 보기 ]                     │
│  [ 다시 하기 ]    [ 로비로 ]            │
└─────────────────────────────────────────┘
```

- 최종 순위 + **자산 총액 공개** (종료 후에만)
- 파산 구제 리필 금액 차감 후 최종 자산 표시
- **MVP 배지 없음** — 단순 자산 순위만 표시
- **[순위 추이 보기]**: 라운드별 플레이어 순위 변동 꺾은선 그래프
  - X축: 라운드 (1~10), Y축: 순위 (1등~N등)
  - 역전 구간, 최종 순위 강조
- 다시 하기 / 로비로 버튼

## 7. 게임 흐름

```
[로비 홈] → 방 만들기(설정) or 방 입장(코드 입력)
    ↓
[대기실] → 플레이어 준비 → 전원 준비 완료 → 방장이 게임 시작
           (설정 변경 시 전원 준비 자동 해제)
    ↓
[게임 시작] 시나리오 생성 + 특수 카드 8장 동일 지급 + 1라운드 이벤트 카드 추첨
    ↓
[1라운드] 드래프트 없음 → 매매 페이즈
          → 전원 레디 or 타이머 만료 → 라운드 결과 화면
    ↓
[2라운드~] 드래프트 (플레이어 수+1장, 꼴지부터 선택) + 매매 동시 진행
           → 전원 레디 or 타이머 만료
           → 라운드 결과 연출 (특수카드 익명 → 라운드카드 → 상장폐지 처리 → 정산 → 순위)
    ↓
[3 / 6 / 9라운드 후] 등수 기반 보너스 카드 차등 지급
                     최하위: 일반 3장 + 특수카드 무효 1장
                     9라운드 꼴등: 라운드 카드 선택권 추가
    ↓
[최종 라운드 반복]
    ↓
[최종 결과] 순위 + 자산액(파산 구제 리필 차감 후) 공개
```

### 8.2 동기화 전략

- 타이머는 서버 타임스탬프(`roundStartAt`) 기준 클라이언트 보정
- 라운드 종료: 방장 트리거 (전원 레디 감지 또는 타이머 만료 시)
- 특수 카드 사용 내역은 라운드 종료 전까지 본인만 열람 가능
- 정보 카드 결과는 로컬 상태로만 표시 (RTDB 기록 없음)
- 라운드 이벤트 카드는 라운드 시작 시 미리 추첨하여 RTDB 저장 (결과 화면에서 공개)
- Firebase RTDB 배열 → 객체 변환 이슈: `Object.values()` 로 정규화 처리

### 8.3 주요 함수 (rtdb.ts)

| 함수 | 역할 |
|------|------|
| `createRoom` | 방 생성, 회사/카드 초기화 |
| `joinRoom` | 방 입장 |
| `startGame` | 게임 시작, 1라운드 이벤트 카드 추첨 |
| `recordTrade` | 매매 기록 + 현금/포트폴리오 즉시 반영 |
| `playSpecialCard` | 특수 카드 사용 기록, 카드 소멸 처리 (플레이어 대상 카드 포함) |
| `useInfoCard` | 정보 카드 소멸 처리 |
| `chooseDraft` | 드래프트 카드 선택 + 핸드 추가 |
| `calculateRoundResult` | 라운드 결과 계산 (방장만) — 세금/소각/구제 정산 포함 |
| `checkDelisting` | 상장폐지 조건 확인 + 신규 회사 등장 처리 |
| `applyBankruptcyRelief` | 파산 구제 발동 + refillTotal 누적 |
| `applyTax` | 현금 보유세 차감 (라운드별 세율 적용) |
| `applyCashBurn` | 현금 소각 적용 대상자 처리 |
| `setRoundReady` | 레디 상태 토글 |
| `endRound` | 라운드 종료 (status → round_result) |
| `nextRound` | 다음 라운드 시작, 카운터 초기화, 보너스 카드 지급, 다음 이벤트 카드 추첨 |
| `dissolveRoom` | 방 데이터 전체 삭제 (방장 전용) → 모든 플레이어 로비로 이동 |
