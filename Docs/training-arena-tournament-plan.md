## 1. 개요

### 1.1 목적
- 예선 → 본선 → 토너먼트 **3단계 대회 구조** 정의
- 각 단계의 **참가 인원 / 진출 규칙 / 대진 생성 방식** 명시
- 게임 시작 시 설정 가능하도록 하되, 본 문서는 **기본값**을 전제로 함

### 1.2 전체 규모 (기본값)

예선은 **등록된 캐릭터 전원이 참가**하며, 총 참가 인원은 모드 설정에 따라 달라진다.

| 단계 | 참가 인원 | 진출 인원 | 대진 방식 |
|------|----------|----------|-----------|
| **예선** | 모드별 가변 (전원 참가) | 32명 | 생존 토너먼트 (32명 남을 때까지 탈락) |
| **본선** | 32명 | 16명 | 4명 × 8개조 더블 엘리미네이션 변형 |
| **토너먼트** | 16명 | 1명 (우승) | 단판 토너먼트 (16강 → 결승) |

### 1.3 예선 참가 인원 (모드별)

| 모드 | 참가 풀 | 예상 인원 |
|------|---------|----------|
| **플레이어 모드** | 오리지널 전체 (본인 포함) | 100명 |
| **시뮬레이션 모드 — 오리지널 전용** | 오리지널 전체 | 100명 |
| **시뮬레이션 모드 — IP 포함** | 오리지널 + 등록된 모든 IP 캐릭터 | 100 + IP 누적 합계 (현재 104 이상) |

> IP 캐릭터 수는 IP 등록이 늘어날수록 증가. 시뮬레이션 모드 IP 토글 ON 시 전부 참가.
> 본선·토너먼트 인원(32 / 16)은 모드와 무관하게 고정.

## 2. 예선전 — 전원 참가 생존 토너먼트

### 2.1 규칙
- **등록된 캐릭터 전원 참가** (모드별 §1.3 참조)
- 매 라운드 1:1 랜덤 매칭, **패자 탈락** (승자 진출)
- **남은 인원이 정확히 32명이 되면 예선 종료**
- 인원이 32의 배수가 아닐 경우 라운드별 탈락 수를 조정 (§2.3 참조)

### 2.2 대진 생성
- 라운드 시작 시 생존자를 전투 시드 기반 Fisher-Yates 셔플
- 인접한 2명을 1:1로 매칭
- 홀수 인원 시 셔플 결과 **마지막 1명 부전승** (자동 진출)
- 시드 편성(상위권 격리) 없음 — 전원 동등 취급

### 2.3 라운드 진행 로직

각 라운드에서 치를 경기 수는 **32명 초과분만 탈락**시키도록 조정.

```
라운드 시작 시:
  생존자 수 = N
  if N ≤ 32:
    예선 종료
  else:
    목표 탈락자 = N - 32
    가능한 최대 경기 = floor(N / 2)
    이번 라운드 경기 수 = min(목표 탈락자, 가능한 최대 경기)
    → 해당 수만큼 매칭 생성, 나머지는 자동 진출
```

### 2.4 라운드 예시

**오리지널 100명 (플레이어 모드 / 시뮬레이션 오리지널)**
| 라운드 | 생존자 | 경기 수 | 자동 진출 | 라운드 후 생존 |
|--------|--------|---------|----------|---------------|
| R1 | 100 | 50 | 0 | 50 |
| R2 | 50 | 18 (목표 탈락 18) | 14 | 32 |
| 합계 | — | **68경기** | — | 32 |

**시뮬레이션 IP 포함 예시 (104명 = 오리지널 100 + 원피스 2 + 블리치 2)**
| 라운드 | 생존자 | 경기 수 | 자동 진출 | 라운드 후 생존 |
|--------|--------|---------|----------|---------------|
| R1 | 104 | 52 | 0 | 52 |
| R2 | 52 | 20 | 12 | 32 |
| 합계 | — | **72경기** | — | 32 |

**대형 풀 예시 (256명)**
| 라운드 | 생존자 | 경기 수 | 자동 진출 | 라운드 후 생존 |
|--------|--------|---------|----------|---------------|
| R1 | 256 | 128 | 0 | 128 |
| R2 | 128 | 64 | 0 | 64 |
| R3 | 64 | 32 | 0 | 32 |
| 합계 | — | **224경기** | — | 32 |

### 2.5 진출자 및 다크호스
- 생존 32명 → 본선 진출
- **예선 탈락자 중 30%** → 다크호스 보상 (+8 랜덤 스탯, 상위 기획서 §6.3)
  - 100명 참가 시 탈락 68명 → 약 20명 다크호스
  - 104명 참가 시 탈락 72명 → 약 22명 다크호스

### 2.6 HP 회복
- 예선 시작 시: HP 전체
- **라운드 사이**: `회복량 = 최대 HP × 10%`, 단 최대 HP 상한
  - 예) 최대 100, 현재 30 → 40 (10 회복)
  - 예) 최대 100, 현재 95 → 100 (상한까지 5만 회복)
- 자동 진출자(부전승)는 HP 변화 없음

### 2.7 시각화
- 플레이어 경기: 자동 전투 연출 (PixiJS)
- 다른 경기: 결과만 즉시 반영 (스킵 권장, 로그는 생성)
- 플레이어 탈락 이후 남은 예선은 스킵 모드 기본

## 3. 본선 — 32명 조별 더블 엘리미네이션 변형

### 3.1 조 편성
- 본선 진출 32명을 **4명 × 8개조 (A~H조)** 로 랜덤 분산
- 편성 시드는 전투 시드와 별개 (재현 가능)

### 3.2 조별 진행 (조당 4명 → 2명 진출)

각 조에서 아래 5경기로 **2명이 토너먼트 진출**.

```
[조별 진행 다이어그램]

1경기  P1 vs P2  →  W1 / L1
2경기  P3 vs P4  →  W2 / L2

[승자전]
3경기  W1 vs W2  →  승자 = 조 1위 (토너먼트 진출 1번)
                     패자 = 최종전 진출

[패자전]
4경기  L1 vs L2  →  승자 = 최종전 진출
                     패자 = 탈락

[최종전]
5경기  (3경기 패자) vs (4경기 승자)
                  →  승자 = 조 2위 (토너먼트 진출 2번)
                     패자 = 탈락
```

### 3.3 경기 순서

| 경기 | 내용 | 승자 | 패자 |
|------|------|------|------|
| 1경기 | P1 vs P2 | W1 → 승자전 진출 | L1 → 패자전 진출 |
| 2경기 | P3 vs P4 | W2 → 승자전 진출 | L2 → 패자전 진출 |
| 3경기 (승자전) | W1 vs W2 | **조 1위 → 토너먼트 진출** | 최종전 진출 |
| 4경기 (패자전) | L1 vs L2 | 최종전 진출 | **조 탈락** |
| 5경기 (최종전) | 승자전 패자 vs 패자전 승자 | **조 2위 → 토너먼트 진출** | **조 탈락** |

### 3.4 조별 경기 수
- 조당 5경기 × 8개조 = **총 40경기**
- 1인 최대 3경기 (1경기 → 승자전 → 최종전 패배 시)
- 1인 최소 2경기 (승자전 승리로 조 1위 확정)

### 3.5 HP 회복
- **본선 시작 시**: HP 전체 회복
- **조 내 경기 사이**: `회복량 = 최대 HP × 10%` (최대 HP 상한)
- **단계 이행 시** (본선 → 토너먼트): 전체 회복

### 3.6 진출자
- 조 1위 8명 + 조 2위 8명 = **16명 토너먼트 진출**
- 조별 탈락자 16명에게는 본선 진출 보상(+3)만 지급

## 4. 토너먼트 — 16명 단판

### 4.1 규칙
- 16명 단판 토너먼트 (16강 → 8강 → 4강 → 결승)
- 총 15경기

### 4.2 대진 편성
- 조 1위 8명과 조 2위 8명을 **1위끼리 or 2위끼리 16강에서 만나지 않도록** 교차 배치
  - 예) 16강 1경기: A조 1위 vs E조 2위
  - 예) 16강 2경기: C조 2위 vs G조 1위
- 브래킷은 시드 기반 결정론으로 생성

### 4.3 라운드별 이름

| 라운드 | 경기 수 | 진출 |
|--------|--------|------|
| 16강 | 8 | 8명 |
| 8강 | 4 | 4명 |
| 4강 | 2 | 2명 |
| 결승 | 1 | 1명 (우승) |

### 4.4 HP 회복
- 라운드 사이: `회복량 = 최대 HP × 10%` (최대 HP 상한)
- (4강 → 결승 특전 회복 없음)

### 4.5 보상 (상위 기획서 §6.3 연계)

| 결과 | 랜덤 스탯 |
|------|----------|
| 우승 (1명) | +10 |
| 준우승 + 4강 탈락 (3명) | +7 |
| 16강·8강 탈락 (12명) | +4 (본선 진출 보상) |

## 5. 전체 경기 수 요약

예선 경기 수는 참가 인원에 따라 가변.

| 단계 | 경기 수 |
|------|---------|
| 예선 | N명 → 32명 축소에 필요한 경기 수 |
| 본선 | 40 (5 × 8조, 고정) |
| 토너먼트 | 15 (고정) |

**대표 시나리오**

| 참가 인원 | 예선 | 본선 | 토너먼트 | 합계 |
|----------|------|------|----------|------|
| 100 (오리지널) | 68 | 40 | 15 | **123** |
| 104 (IP 포함 초기) | 72 | 40 | 15 | **127** |
| 200 (IP 확장) | 168 | 40 | 15 | **223** |
| 256 | 224 | 40 | 15 | **279** |

시뮬레이션 모드에서는 이 전체가 **1회차**이며, 배치로 빠르게 실행 (1000경기/초 목표).

## 6. 대진 생성 알고리즘

### 6.1 셔플
```
1. 참가자 배열 가져오기
2. 전투 시드(대회 ID)로 Fisher-Yates 셔플
3. 각 단계별 규칙에 따라 자리 배정
```

### 6.2 결정론
- 동일 대회 ID + 동일 참가자 목록 → 동일 대진
- 리플레이·디버그·통계 비교에 필수

### 6.3 편향 방지
- 조 편성 시 같은 아키타입이 한 조에 몰리지 않도록 **약한 밸런싱** 옵션
  - 기본은 완전 무작위, 밸런싱은 설정 토글
- 토너먼트 대진은 1위/2위 교차 배치 강제

## 8. Firestore 스키마 (대회 관련)

```
/tournaments/{tournamentId}
  - round: number (회차 번호)
  - seed: number
  - stage: "예선" | "본선" | "토너먼트" | "완료"
  - participants: charId[]        (64명)
  - qualifier: charId[]           (예선 통과 32명)
  - groupA..H: {
      players: charId[4],
      rank1: charId,              (조 1위)
      rank2: charId               (조 2위)
    }
  - finalists: charId[]           (토너먼트 진출 16명)
  - winner: charId
  - darkhorses: charId[]          (다크호스 약 10명)
  - createdAt: timestamp          (ISO 8601)

/tournaments/{tournamentId}/matches/{matchId}
  - stage: "예선" | "본선" | "토너먼트"
  - round: number
  - groupId: string | null        (본선만)
  - attacker, defender, winner: charId
  - seed, totalTurns: number
  - createdAt: timestamp
```

전투 로그는 [전투 시뮬레이션 기획서 §12](./training-arena-battle-plan.md) `/matches/{matchId}/log` 서브컬렉션 활용.
