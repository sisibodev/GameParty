## 1. 개요

본 문서는 `육성 전투 시뮬레이션` 게임에 등장하는 **전체 캐릭터 풀**과 각 캐릭터의 **기본 전투 스탯 배분** 및 **외형 묘사**를 데이터 표 중심으로 정리한다.

- **오리지널 캐릭터** — 자체 제작, 플레이어 모드 + 시뮬레이션 모드 모두 사용
- **IP 캐릭터** — 외부 IP(만화 / 게임 등)에서 차용, **시뮬레이션 모드 전용**

---

## 2. 캐릭터 번호 체계

### 2.1 오리지널 캐릭터 (번호 1 ~ 999)

- **범위**: `1` ~ `999`
- 순차 부여 (1번부터)
- 기획 규모: **약 100명**
- 플레이어 모드 + 시뮬레이션 모드 모두에서 사용 가능

### 2.2 IP 캐릭터 (번호 1000 이상)

- **범위**: `1000` 이상
- **IP 번호 체계**로 구분
- **시뮬레이션 모드 전용** (플레이어 모드 선택 불가)

### 2.3 IP 번호 규칙

각 IP(원피스, 블리치 등)에 **100 단위 IP 번호**를 부여하고,
`1000 + IP번호 + IP 내 인덱스(0~99)` 형식으로 캐릭터 번호를 산출한다.

**공식**
```
캐릭터 번호 = 1000 + IP번호 + 인덱스(0 ~ 99)
```

**예시**
- 원피스 (IP 100)
  - 1100 = 루피 (인덱스 0)
  - 1101 = 조로 (인덱스 1)
  - 1102, 1103 … (추가 캐릭터)
- 블리치 (IP 200)
  - 1200 = 이치고 (인덱스 0)
  - 1201 = 루키아 (인덱스 1)
  - 1202, 1203 … (추가 캐릭터)

### 2.4 IP 레지스트리 (등록 IP 목록)

| IP 번호 | IP 이름 | 캐릭터 번호 범위 | 비고 |
|------|------|------|------|
| 100 | 원피스 | 1100 ~ 1199 | 애니 / 만화 |
| 200 | 블리치 | 1200 ~ 1299 | 애니 / 만화 |
| 300 | (예약) | 1300 ~ 1399 | — |
| 400 | (예약) | 1400 ~ 1499 | — |
| … | … | … | 이후 100 단위 순차 부여 |

### 2.5 IP 확장 규칙

- 신규 IP 추가 시 **다음 100 단위 IP 번호** 부여 (300, 400, 500 …)
- IP당 최대 **99명**까지 등록 (00 ~ 99)
- 한 IP가 99명 초과 시 → 해당 IP에 **추가 IP 번호** 별도 발급 (예: "원피스 확장 = IP 900")

## 3. 스탯 배분 규칙

> 상위 규칙: [메인 기획서 §4](./training-arena-plan.md)

### 3.1 baseCombat 필드 구조 (v0.3, 물리·마법 분리)

`characters.json`의 각 캐릭터 `baseCombat` 객체:

| 필드 | 의미 | 비고 |
|------|------|------|
| `maxHp` | 최대 체력 | |
| `pAtk` | 물리 공격력 | warrior/berserker/assassin/ranger/tank/paladin 주력 |
| `mAtk` | 마법 공격력 | mage/support 주력 |
| `pDef` | 물리 방어력 | |
| `mDef` | 마법 방어력 | |
| `spd` | 속도 | |
| `crit` | 치명타율 | |
| `eva` | 회피율 | |
| `maxMana` | 최대 마나 | |
| `acc` | 명중률 | 기본값 80 (모든 캐릭터 공통) |
| `aggression` | 스킬 사용 적극성 | |

> v0.3 변경: 기존 `atk` → 직군에 따라 `pAtk`/`mAtk` 분리. 기존 `def` → `pDef`/`mDef` 분리. `acc` 신설.
> 마법사/서포터 계열(mage/support): `mAtk` 주력, `pAtk = floor(mAtk × 0.4)`.
> 그 외(warrior/berserker/assassin/ranger/tank/paladin): `pAtk` 주력, `mAtk = floor(pAtk × 0.4)`.

### 3.2 캐릭터별 스탯 배분 방향

캐릭터마다 **컨셉과 외형에 맞추어 baseCombat을 배분**한다.

- 각 캐릭터의 구체 스탯은 `characters.json` 참조
- 아키타입별 직군 정체성은 `pAtk`/`mAtk` 비율로 자연 표현
- 실제 전투 스탯은 `statDeriver.ts`가 성장 스탯(VIT/STR/AGI/INT/LUK)을 더해 최종 산출

## 7. 데이터 저장 구조 (초안)

Firestore 컬렉션 예시:

```
characters/
  {characterId}
    id: number              // 1~999 (오리지널) 또는 1000+ (IP)
    name: string            // 사람 이름 (한국어 발음 표기, v0.2+)
    title: string           // 직업명 (v0.2+, 구 이름)
    description: string     // 외형 묘사
    archetype: string       // tank/warrior/berserker/assassin/ranger/mage/support/paladin
    ipId: number | null     // null=오리지널, 숫자=IP 번호
    ipName: string | null   // 표기용
    baseCombat: {           // v0.3+: 물리·마법 분리 (구 statBonus 대체)
      maxHp: number
      pAtk: number          // 물리 공격력
      mAtk: number          // 마법 공격력
      pDef: number          // 물리 방어력
      mDef: number          // 마법 방어력
      spd: number
      crit: number
      eva: number
      maxMana: number
      acc: number           // 명중률 (기본 80)
      aggression: number
    }
    skillIds: string[]      // 고정 배정 스킬 3종 ID (v0.5.0+)
    selectable: {
      player: boolean       // 플레이어 모드 선택 가능 여부
      simulation: boolean   // 시뮬레이션 모드 참가 가능 여부
    }
```

**정합성 검증 규칙**
- `id < 1000` ⇒ `ipId === null`, `selectable.player === true`
- `id >= 1000` ⇒ `ipId !== null`, `selectable.player === false`
