# 🎬 배틀 그랑프리 — 시뮬레이션 모드 기획서

> 버전: v0.2 (코드 기반 확정, 2026-04-25 검증)
> 상위 문서: [training-arena-plan.md](./training-arena-plan.md)
> 연관 코드: `src/games/training-arena/pages/SimulationPage.tsx`

---

## 1. 목적

플레이어 개입 없이 **AI 캐릭터끼리 토너먼트를 관전**하는 모드.
본 게임(플레이어 모드)와 **별도 로직**으로 유지되며, v0.4 본 게임 리디자인(전술 카드, 아이템, 골드, 스킬 학습 등)의 영향을 받지 않는다.

> **원칙**: 본 게임 로직이 어떻게 바뀌어도 시뮬레이션 모드는 현재 밸런스·규칙으로 계속 관전 가능해야 한다.

---

## 2. 진입점

- `SlotSelectPage` 또는 메인 메뉴에서 "시뮬레이션 모드" 버튼.
- `GamePhase = 'simulation'`으로 전환, 단독 화면으로 동작.
- 슬롯 시스템과 독립 — 플레이어 세이브와 섞이지 않음.

---

## 3. 설정 화면 (시작 전)

### 3.1 캐릭터 풀 선택
- **오리지널만**: 오리지널 100명 중 랜덤 추첨.
- **IP 캐릭터 포함**: 오리지널 + IP 캐릭터 풀 전체에서 추첨.
  - 본 게임(플레이어 모드)은 오리지널 전용이지만, 시뮬레이션에서는 IP 캐릭터를 관전 목적으로 허용.

### 3.2 시드 입력 (선택)
- 시드 고정 시 동일한 결과 재현.
- 비워두면 랜덤 시드.

### 3.3 실행 버튼
- "시뮬레이션 시작" → 예선/본선/토너먼트 자동 진행.

---

## 4. 실행 중 화면

### 4.1 라운드·경기 진행 시각화
- 현재 라운드/단계/경기 번호 상단 고정 표시.
- 진행 바 or 카운터로 전체 진행률 표시.

### 4.2 기록 창 (v0.4.0 추가)
라운드 진행 중 아무 때나 **기록 창**을 열어 현재까지의 성적을 확인.

표시 내용:
- **통산 전적** — 해당 회차 시뮬 전체에서 각 캐릭터의 누적 승/패
- **최고 기록** — 이번 시뮬에서 최다 연승·최다 승 캐릭터 등
- **상대 전적** — 캐릭터 간 맞대결 기록

> 기존에는 라운드 종료 시에만 열람 가능 → 라운드 도중 언제든 열람 가능하도록 개선.

### 4.3 경기 결과 스낵바
- 매 경기 종료 시 승자/패자를 하단 토스트로 짧게 표시.

---

## 5. 중단 및 재개

- 시뮬레이션 진행 중 사용자가 나가기를 누르거나 페이지를 벗어나도, **리셋이 아닌 이상 상태는 유지**.
- 다시 시뮬레이션 모드로 들어오면 **중단된 지점부터 재개** 가능.
- "처음부터 다시" 버튼으로 명시적 리셋 가능.
- 리셋 조건:
  - 사용자가 명시적 "리셋" 버튼 클릭
  - 캐릭터 풀 설정 변경(오리지널↔IP 포함 전환)
  - 시드 변경

---

## 6. 종료 화면

### 6.1 우승자 발표
- 우승 캐릭터 이름·아케타입·최종 스탯 표시.
- 결승 로그 재생 옵션.

### 6.2 전체 기록
- 라운드별 대진표와 경기 로그.
- 통산 전적·최고 기록·상대 전적 최종 버전.
- 경기 클릭 → 해당 경기의 턴별 로그 재생(리플레이).

### 6.3 공유·내보내기 (향후)
- 시드 + 결과 요약을 텍스트로 복사.

## 7. 본 게임과의 경계 (v0.4.2 기준)

| 요소 | 본 게임(플레이어 모드) | 시뮬레이션 모드 |
|------|---------------------|----------------|
| 초기 스탯 | 플레이어 1 / NPC 10 | 전원 NPC_BASE_GROWTH 동일 |
| 가챠 | 라운드당 1장 (범위: C~SSS) | 기존 100연 가챠 고정값 방식 |
| 전술 카드 | ✓ 사용 (8종 선택) | 미사용 |
| 아이템·상점·골드 | ✓ 사용 | 미사용 |
| 스킬 학습 (턴 기반) | ✓ 사용 (등급별 차등) | 미사용 (기존 랜덤) |
| 라이벌 시스템 | ✓ 사용 (+5 누적) | 미사용 |
| 본선 구조 | ✓ 더블 엘리미네이션 (조별) | ⏳ 미확인 (기존 방식?) |
| 캐릭터 해금 | ✓ 사용 | 영향 없음 (전체 풀 관전) |
| 저장소 | Firestore | IndexedDB + 세션 상태 |

**원칙**: 시뮬레이션은 독립 로직 유지. 본 게임 변경 시 회귀 없이 동작해야 함.

## 8. UI/UX 구성 요소

- **나가기 버튼**: 우상단 고정 ✕, 누르면 "메인으로 나가시겠습니까?" 확인. (본 게임과 동일 패턴)
- **배속 컨트롤**: 전투 배속(1x/2x/4x/8x) 독립 조작.
- **스킵**: 현재 경기 즉시 종료, 결과만 표시.
- **로그 토글**: 턴별 로그 on/off.
