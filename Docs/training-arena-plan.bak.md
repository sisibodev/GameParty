## 1. 게임 개요

### 한 줄 설명
캐릭터 하나를 골라 스탯을 찍고, 가챠로 다른 경쟁자들을 강화시킨 뒤, 예선부터 토너먼트까지 올라가 우승을 노리는 **육성형 자동 전투 시뮬레이션**.

### 핵심 특징
- 100% 자동 전투 — 플레이어는 **육성 / 분배 / 선택**만 담당
- 가챠로 뽑은 카드가 **경쟁자들까지 강화**시키는 독특한 구조 (본인 캐릭터가 뽑히면 본인도 강해짐)
- 매 회차마다 우승한 캐릭터의 스킬을 흡수하며 **장기적 성장** 누적
- 플레이어 개입 없이 AI끼리 돌리는 **순수 시뮬레이션 모드** 별도 제공

### 장르
육성 / 자동 전투 / 가챠 / 토너먼트 시뮬레이션

### 플레이 인원
싱글 (1인)

### 렌더링 방식
PixiJS (2D) — 캐릭터 표시, 전투 연출, 가챠 연출

### 서버 방식
없음 (클라이언트 완결)

### 기록 저장
**Firestore** — 플레이어 프로필, 누적 스탯, 보유 스킬, 우승까지 걸린 회차, 대회 이력

## 2. 게임 모드

### 2.1 플레이어 모드
본인 캐릭터를 선택해서 육성하고 우승을 노리는 메인 모드.

### 2.2 시뮬레이션 모드
플레이어 개입 없이 AI 캐릭터끼리만 대회를 돌리는 관전 모드.
- 캐릭터 분배 / 가챠 / 대회 전부 자동
- 결과 / 통계 관찰용

---

## 3. 핵심 게임 루프

```
[플레이어 모드]
캐릭터 선택
  → 초기 스탯 분배 (플레이어 기본 스탯 포인트 지급)
  → 가챠 100연 (경쟁자 + 본인 강화)
  → 예선 → 본선 → 토너먼트
  → 보상 수령 (단계별 랜덤 스탯)
  → 플레이어 추가 스탯 +10 분배
  → 우승 캐릭터 스킬 1개 습득
  → 다음 회차 (누적 진행)
```

목표: **최소 회차로 우승하기** (회차 수를 DB에 기록)

## 4. 스탯 시스템

### 4.1 성장 스탯 (5종)

모든 캐릭터 공통, 초기값 **1로 통일**. 가챠 / 대회 보상으로 상승.

| 스탯 | 영향 |
|------|------|
| 체력 | HP, 방어력, 마나 리젠률 |
| 힘 | 공격력, 방어력 |
| 민첩 | 공격속도, 회피, 크리티컬 |
| 지력 | 마나, 스킬 쿨타임, 마나 리젠률 |
| 행운 | 회피, 크리티컬, 스킬 발동률, 적극성 |

- **최대치**: 각 스탯 **9999**
- 랜덤 부여 시 해당 스탯이 최대치를 초과하면 **다른 스탯으로 재분배**

### 4.2 전투 스탯 (성장 스탯으로부터 파생)

| 전투 스탯 | 계산 기반 |
|------|------|
| HP | 기본값 + 체력 (0이 되면 패배) |
| 공격력 | 기본값 + 힘 |
| 방어력 | 기본값 + 힘 + 체력 |
| 공격 속도 | 기본값 + 민첩 (행동 순서 결정) |
| 크리티컬 | 기본값 + 행운 + 민첩 |
| 회피 | 기본값 + 행운 + 민첩 |
| 마나 | 기본값 + 지력 (스킬 소모 자원, 리젠률은 체력+지력) |
| 적극성 | 기본값 + 행운 (내 턴에 스킬을 사용할지 결정) |

### 4.3 캐릭터 기본 전투 스탯 (100점 베이스)

모든 캐릭터는 아래 기본값에 **추가 100점을 컨셉에 맞게 배분**해서 시작.

| 스탯 | 기본값 |
|------|------|
| HP | 20 |
| 공격력 | 10 |
| 방어력 | 5 |
| 공격 속도 | 1 |
| 크리티컬 | 10 |
| 회피 | 0 |
| 마나 | 10 |
| 적극성 | 5 |

> 예: 중장갑 캐릭터라면 HP / 방어력 쪽에 많이 배분.
> 구체 배분표는 [캐릭터 기획서](./training-arena-character-plan.md) 참조.

## 5. 가챠 시스템

### 5.1 기본 규칙
- 대회 시작 **직전**에 가챠 진행
- **100연 가챠** (연차 수는 밸런스 단계에서 재검토)
- 카드 1장마다 **랜덤 등급** 결정 → 등급에 따라 해당 캐릭터에게 **랜덤 스탯 증가**

### 5.2 등급 테이블

| 등급 | 증가 스탯 | 출현 확률 |
|------|------|------|
| C | +1 | 가장 높음 |
| B | +2 | ↑ |
| A | +3 | ↑ |
| S | +4 | ↑ |
| SS | +5 | ↑ |
| SSS | +6 | 가장 낮음 |

> 확률 수치 구체화는 밸런스 단계(TODO).

### 5.3 본인 캐릭터 규칙
- 가챠에서 **본인 캐릭터가 뽑히면** → 본인 캐릭터도 해당 등급만큼 스탯 상승
- 즉, 가챠는 "경쟁자 강화 vs 본인 강화"의 운 요소

---

## 6. 대회 구조

### 6.1 진행 단계
```
예선 → 본선 → 토너먼트 → 우승
```

### 6.2 HP 회복 규칙
- 각 **단계 이행 시**: HP 최대치 회복
- 단계 내 **경기와 경기 사이**: HP 10% 회복

### 6.3 단계별 보상

| 결과 | 랜덤 스탯 부여 |
|------|------|
| 우승 | +10 |
| 토너먼트 진출 | +7 |
| 본선 진출 | +3 |
| 예선 탈락자 중 **30%** (다크호스 룰) | +5 |

- 추가로 **플레이어 본인에게는 원하는 곳에 분배 가능한 스탯 +10** 별도 지급

## 7. 스킬 시스템 (개요)

> 상세는 [스킬 기획서](./training-arena-skill-plan.md) 참조

- 전체 스킬 풀 약 **30종**
- 캐릭터당 **3개**씩 랜덤 배분
- 플레이어도 시작 시 기본 3개
- 대회 **우승 / 상위 캐릭터의 스킬 중 1개를 플레이어가 습득** → 장기 누적 성장

---

## 8. 전투 시스템 (개요)

> 상세는 [전투 시뮬레이션 기획서](./training-arena-battle-plan.md) 참조

- 자동 턴제 전투 (공격속도 기반 행동 순서)
- 매 턴 스킬 사용 여부는 **적극성 + 마나** 체크
- HP 0 → 패배
- 1:1 또는 다수전 여부는 전투 기획서에서 결정

---

## 9. 캐릭터 (개요)

> 상세는 [캐릭터 기획서](./training-arena-character-plan.md) 참조

- 여러 **직군 (클래스)**를 가진 캐릭터 풀
- 각 캐릭터는 컨셉에 맞게 기본 전투 스탯 100점 배분 + 랜덤 스킬 3개 보유
- 플레이어는 시작 시 하나 선택

## 11. 저장 데이터 (Firestore 개요)

- 플레이어 프로필
  - 선택 캐릭터 ID
  - 누적 성장 스탯 (체력 / 힘 / 민첩 / 지력 / 행운)

- 대회 이력
  - 예선 / 본선 / 토너먼트 결과
- 시뮬레이션 모드 기록 (선택 구현)

> 스키마 확정은 개발 착수 시점에 별도 정의.
