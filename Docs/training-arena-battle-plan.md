## 1. 개요

### 1.1 목적
- 자동 전투의 **결정론적 규칙** 정의
- 스탯 + 스킬 + 확률 판정이 어떻게 승패로 이어지는지 명확화
- 시각화(PixiJS)와 로그 기록의 기반 제공

### 1.2 기본 원칙
- 100% **자동 전투** (플레이어 개입 없음)
- 턴제 (공격속도 기반 행동 순서)
- **시드 기반 결정론** — 동일 시드면 동일 결과 (리플레이·디버그 목적)
- 전투 로그는 매 액션 단위로 기록

## 2. 전투 구조

### 2.1 전투 형식
- **1:1 단판** (기본) — 예선 / 본선 / 토너먼트 공통
- 다수전은 스킬의 `전체 적` 대상에만 영향 (2인전이어도 단일 타겟)
- 무승부 방지 규칙: 50턴 초과 시 **HP% 높은 쪽 승리**, 동률이면 공격력 높은 쪽

### 2.2 전투 흐름

```
[전투 시작]
  ↓
1. 초기화
   - HP/마나 최대치로 세팅
   - 버프/디버프 초기화
   - 쿨타임 0으로 리셋
  ↓
2. 선공 판정 (공격속도 비교, 동률 시 행운)
  ↓
3. 턴 루프 (턴 번호 증가)
   ├─ 턴 시작 처리 (DoT/HoT, 쿨타임 -1, 버프 지속시간 -1)
   ├─ 행동 페이즈 (스킬 or 일반 공격)
   ├─ 피격 처리 (회피/크리/반사 판정)
   ├─ 사망 체크 (HP ≤ 0)
   └─ 다음 행동자로 전환
  ↓
4. 종료 조건 충족
   - 한쪽 HP 0 → 상대 승리
   - 50턴 초과 → HP% 비교
  ↓
[전투 종료]
  - 결과 기록 (Firestore)
  - HP 10% 회복 (다음 경기 전)
```

## 3. 행동 순서 (턴 시스템)

### 3.1 행동 게이지 방식
```
ATB (Active Time Battle) 유사 방식
- 모든 캐릭터가 게이지 0에서 시작
- 매 tick마다 게이지 += 공격속도
- 게이지 100 도달 시 해당 캐릭터 행동
- 행동 후 게이지 0으로 리셋
```

### 3.2 공격 속도 영향
- 공격속도 10 → 게이지 100까지 10 tick
- 공격속도 20 → 5 tick (2배 행동)
- 버프 `시간 가속` 등은 tick 1회당 += 2× 공격속도

### 3.3 동시 발동 처리
- 같은 tick에 여러 캐릭터가 100 도달 → 행운 높은 순
- 행운 동률 → 전투 시드 기반 랜덤

## 4. 액션 유형

### 4.1 일반 공격
- 조건: 마나 부족 or 쿨타임 대기 or 적극성 판정 실패
- 데미지: `공격력 × 1.0` (방어력/크리/회피 반영)
- 마나 회복: 최대 마나 × (지력 + 체력) × 0.001 (턴당)

### 4.2 스킬 사용
- 조건: 마나 ≥ cost, 쿨타임 0, 적극성 판정 통과
- 발동률: `50% + 적극성 × 0.5%` (최대 100%)
- 효과는 스킬 기획서 §4 참조

### 4.3 방어 (자동)
- 별도 액션 아님 — 피격 시 방어력/회피로 반영

---

## 5. 피격 판정 순서

```
1. 명중 판정
   명중률 = 100 - (대상 회피 - 시전자 명중 보정)
   (기본 명중 보정 = 민첩 × 0.5)
   실패 시 "회피" 로그, 데미지 0
  ↓
2. 크리티컬 판정
   크리율 = 시전자 크리 + 민첩 × 0.3 - 대상 회피 × 0.3
   성공 시 데미지 × (1.5 + 행운 × 0.001), 최대 3.0배
   예) 행운 0 → 1.5배 / 행운 500 → 2.0배 / 행운 1500 → 3.0배 (상한)
  ↓
3. 데미지 계산 (퍼센트 감쇠 방식)
   최종 데미지 = 공격력 × 배율 × 100 / (100 + 방어력)
   예) atk 200, def 100 → 100 (50% 감소)
   예) atk 200, def 200 → 67 (67% 감소)
   예) atk 200, def 500 → 33 (83% 감소)
   최소 1 보장 (방어력 무한 증가해도 0이 되지 않음)
  ↓
4. 방어/실드 감쇠
   - 방어 버프(+% 감소) 순차 적용
   - 실드 있으면 실드 먼저 소모
  ↓
5. 반사/반격 처리
   가시 방패 등 반사 딜은 원천 데미지의 n%로 시전자에게 되돌림
  ↓
6. HP 차감 + DoT 갱신
```

## 6. 상태이상 / 버프 시스템

### 6.1 중첩 규칙
| 유형 | 같은 효과 중첩 | 다른 효과 |
|------|----------------|----------|
| 버프 | **덮어쓰기 (지속시간 갱신)** | 공존 |
| 디버프 | **덮어쓰기 (더 강한 값 우선)** | 공존 |
| DoT/HoT | **스택 (최대 5)** | 공존 |

### 6.2 지속시간 감소 타이밍
- 턴 시작 시 -1
- 0 도달 → 해당 턴 효과까지 발동 후 제거

### 6.3 해제 조건
- `완전 회복` 같은 정화 스킬
- 전투 종료 (다음 경기로 이월되지 않음)

---

## 7. 마나 시스템

### 7.1 초기 마나
- 전투 시작 시: 최대 마나 × 50%

### 7.2 턴당 회복
```
마나 회복 = 최대 마나 × (체력 + 지력) × 0.001
예) 최대 마나 200, 체력 50, 지력 100 → 턴당 30 회복
```

### 7.3 스킬 사용 실패
- 마나 부족 시 스킬 슬롯 스킵 → 일반 공격
- 쿨타임 중 스킬은 선택 풀에서 제외

---

## 8. 적극성과 스킬 선택

### 8.1 적극성 판정
- 매 턴 행동 시점에 확률 판정
- 성공 → 스킬 사용 시도 / 실패 → 일반 공격

### 8.2 스킬 선택 알고리즘
```
사용 가능 스킬 = 쿨타임 0 + 마나 충족
  ↓
priority 낮은 순 정렬
  ↓
동일 priority 내에서 가중치 랜덤:
  - 일반: 1
  - 희귀: 2
  - 영웅: 4
  - 전설: 8
  ↓
선택된 스킬 발동
```

### 8.3 조건부 트리거
- `불굴의 의지`: HP 20% 이하 시 자동 발동 (적극성 무관)
- `반격 태세`: 피격 시 조건 충족하면 자동 반격
- 조건부 스킬은 priority 0으로 최우선 처리

## 9. 결정론과 시드

### 9.1 시드 사용 범위
- 전투 시작 시 시드 생성 (대회 ID + 경기 번호)
- 모든 랜덤(명중/크리/적극성/스킬선택)에 동일 RNG 사용
- 리플레이·디버그 시 동일 시드로 재현

### 9.2 RNG 구현
- xorshift32 또는 Mulberry32 등 가벼운 PRNG
- 전역 `Math.random` 사용 금지

## 11. 시각화 (PixiJS)

### 11.1 연출 요소
- 캐릭터 스프라이트 2체 좌우 배치
- HP/마나 바 (상단)
- 스킬 사용 시 이펙트 + 스킬명 팝업
- 데미지 숫자 플로팅 (크리는 더 크게)
- 버프/디버프 아이콘 스택

### 11.2 재생 속도
- 기본 1x / 2x / 4x / 스킵 (즉시 결과)
- 시뮬레이션 모드는 스킵 기본값

## 13. 밸런스 튜닝 포인트

| 변수 | 기본값 | 영향 |
|------|--------|------|
| 방어력 공식 | 100 / (100 + def) | 퍼센트 감쇠 — 후반 스탯에서도 선형성 유지 |
| 크리 배율 | 1.5 + 행운 × 0.001 (cap 3.0) | 행운 스탯이 확률 + 배율에 이중 작용 |
| 마나 회복 계수 | 0.001 | 낮추면 스킬 사용 간격 증가 |
| 최대 턴 수 | 50 | 장기전 방지 |
| 적극성 계수 | 0.5% | 높이면 스킬 난무 |
